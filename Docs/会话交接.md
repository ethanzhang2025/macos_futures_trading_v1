## 一、项目定位

- **什么**：macOS 原生期货交易终端 **Alpha 样机**（第 1-30 天阶段）
- **技术栈**：Swift 6 / SwiftUI / AppKit / SPM（根 Package.swift 是库层，`App/Package.swift` 是可执行 target）
- **交付定义**：能展示「这是一个真正为 Mac 做的专业期货终端」
- **数据源**：新浪期货 API（quote + K 线，无需鉴权，Referer 即可）
- **交易撮合**：Mock（300ms 延时回成交），不做真实交易牌照
- **开发/运行分离**：Linux 机开发 + commit + push；Mac 机 pull + `swift run` 验证（SwiftUI 只在 Mac 可编译 App 层）

**已做的 3 次修复尝试**（按时间顺序）：

1. `de6457c` 修复：多窗口关闭时崩溃
   - `AppDelegate.cancellables` 全局数组 → `windowCancellables: [ObjectIdentifier: AnyCancellable]`（按窗口分组，关窗精确清理）
   - `AppViewModel.startPolling` 的 `Task { ... }` 加 `[weak self]`，防止 Task 强持有 vm

2. `9239b1d` 修复：多窗口关闭崩溃（显式 teardown 顺序）
   - `AppDelegate` 加 `windowViewModels: [ObjectIdentifier: AppViewModel]`
   - `windowWillClose` 里按顺序：①`vm.stopPolling()` → ②清 title Combine 订阅 → ③移除 vm/window 强引用 → ④`removeObserver`

## 七、关键设计决策（已冻结，勿推翻）

- **Alpha 不接 CTP**：Mock 足够，真实接入放 Beta
- **Alpha 不上 Metal 图表引擎**：Canvas 满足 500-2000 bars 需求；Metal 的收益要到 L2 深度/tick tape 才显现，放 Beta
- **不为"未来换 Metal"做抽象层**：换时新起 `MetalKLineView` 平替
- **OI 副图「放弃」已翻盘**：数据源升级后 `SinaKLineBar.openInterest` 有值，已做完
- **真实主力切换（阶段 4）暂不做**：新浪 API 月份合约覆盖不完整（RB2510 空、AU2506 过期数据、IF2506 空），做了演示效果不稳
- **金融期货 parse 是近似**：非交易时段 close=last 可接受，交易时段字段顺序需进一步核实
- **watchList 窗口间不实时同步**：Alpha 可接受（UserDefaults load + didSet 写，窗口间要重启看最新）
