# 产品设计书 · 中国期货 Mac/iPad 原生交易终端

> 这是项目的主产品设计文档（Master Product Spec）。D1/D2/D3 是基础层；本文档是数据驱动的产品+商业论证，用于团队对齐、投资人沟通、合伙人尽调。

## 摘要

**产品一句话**：为专业交易者打造的 Mac/iPad 原生**专业工作台** —— 盯盘、预警、复盘、训练、日志，让你盯得更快、复得更清、练得更稳。中国期货版 **TradingView × Linear × iPad Pro**。

**定位澄清**：Stage A 是**专业工作台**（补充主交易软件的工作流），**不是实盘交易终端**。Stage B 加入 CTP 下单后成为完整终端。

**核心数字卡**：

| 维度 | 数字 | 依据 |
|------|------|------|
| TAM（Mac 期货 Pro 级）| ≈ **6 万人** / 年付费容量 **¥2400 万** | 250 万活跃 × 8% Mac × 30% 付费意愿 |
| SAM（10 年可及）| ≈ **¥720 万** ARR | TAM 的 30% |
| Stage A M9 目标 | **500 Pro / 月流水 ¥1.7 万** | |
| Stage B M24 目标 | **1 万付费 / ARR ¥2000 万+** | |
| 启动资金 | **¥20 万**（2 人 6-9 月跑道）| |
| 生死节点 | **M6 上线收钱** | 未上线 = 项目破产 |
| 盈亏平衡 | **≈ M8-M9** | 月流水覆盖月成本 |

**三阶段战略**：
- **Stage A（M0-M9）** · 图表+复盘工具，不接 CTP，**活下来**
- **Stage B（M12-M24）** · CTP 下单 + 麦语言完整兼容 + iPad 专业，**做大**
- **Stage C（M24+）** · 据结果定（Windows/证券/出海/策略市场）

**决胜公式**：Mac 原生审美 × Apple 生态协同 × 麦语言兼容 × 透明订阅 = 拆除文华护城河

## 二 · 市场与用户

### 2.1 市场规模（TAM / SAM / SOM）

```
中国期货活跃账户数（2024 估）≈ 250 万
├─ Mac 用户渗透率 ≈ 8%           → Mac 期货用户 ≈ 20 万
├─ 其中专业付费意愿 ≈ 30%         → Pro 级 Mac 用户 ≈ 6 万
└─ 按 ¥399/年定价                → TAM = ¥2400 万/年
```

**敏感性分析**（最悲观下仍能支撑小而美生意）：
- Mac 渗透率 5%（而非 8%）→ TAM 降 37% = ¥1500 万
- 付费意愿 15%（而非 30%）→ TAM 降 50% = ¥1200 万
- 组合最坏 → **TAM ≈ ¥750 万 —— 仍能支撑 ¥300-500 万 ARR 目标**

**SAM（10 年可及，30% 渗透）**：¥720 万 ARR

**SOM（分阶段目标）**：
- M9：500 人 / ¥20 万 ARR（Stage A 达标）
- M18：5000 人 / ¥200 万 ARR
- M24：10000 人 / ¥2000 万+ ARR（含 Pro Max 升级 + 券商代付）

## 三 · 产品功能

### 3.0 期货机制速查（内部对齐用）

做 Mac 期货终端必须理解这些机制 —— 很多功能的"必须性"来自这些规则，不是产品偏好。

**交易机制 · 期货 vs 股票**

| 机制 | 期货 | 股票 | 我们必须支持的 UI/逻辑 |
|------|-----|------|--------------------|
| 方向 | **双向**（可做多/做空）| 仅做多 | 多/空双按钮 + 双向持仓展示 |
| 交割 | **T+0** 当日平仓 | T+1 次日卖 | 当日平仓提示 + 强平时间提醒 |
| 杠杆 | **保证金** 10-15% | 无 | 保证金占用 + 可用资金实时 |
| 时段 | 日盘 9:00-15:00 **+ 夜盘 21:00-02:30** | 仅日盘 | 夜盘标识 + 隔夜 K 线逻辑 |
| 结算 | **每日结算价**（非收盘）| 无 | 结算价线显示在图表 |
| 开平仓 | **区分开仓/平今/平昨**（手续费不同）| 无 | 订单票据显示开平属性 |
| 涨跌停 | **动态**（不同合约不同）| 固定 10% | 实时读取 + 提醒 |
| 持仓 | 同合约**可同时多空**（部分品种）| 无 | 双向持仓 UI |

**数据特征**（影响 Stage B 后端架构设计）

| 维度 | 估算值 | 设计含义 |
|------|------|--------|
| 日 Tick 数据量 | ~500 MB - 1 GB/天 | 必须压缩存储（LZ4 3-5x）|
| 年 Tick 数据量 | ~375-750 GB/年 | 分片 + 冷热分层 |
| K 线多级合成 | Tick → 1m → 5m → 15m → 1h → 日 | 内存常驻 + 懒加载 |
| 断线重连 | 日盘夜盘切换频繁 | 需心跳 + 指数退避（见 D3）|

### 3.2 核心模块（12 个）

**① Metal 自研图表引擎**（核心差异化）
- 性能基线：**10 万 K 线 60fps / 冷启动 < 1s / 交互延迟 < 16ms**
- 对手基线：文华 30fps、快期 WebView 卡顿

**② 指标库 56 个 v1**
- 趋势 10 / 震荡 12 / 量价 8 / 波动率 8 / 支撑阻力 6 / **期货特有 12**（差异化）
- 期货特有 12 个（持仓量/基差/价差等）**TradingView 没有**

**③ 画线工具 6**
- 趋势 / 水平 / 矩形 / 平行通道 / 斐波那契 / 文字

**④ 复盘分析 8 张图**
- 差异化：**品种热力矩阵 + 时段分析** 文华/快期都没做好

**⑤ K 线回放**（Stage A 新增 · 沉浸式复盘）
- 选历史日期 + 品种，任意周期回放
- 播放 / 暂停 / 加速（2x-8x）/ 倒退 / 单步
- 当日成交点叠加标注 —— 替代纯报表复盘

**⑥ 条件预警中心**（Stage A 新增 · 专业高频刚需）
- 价格 / 画线 / 波动率 / 成交量异常预警
- 通知渠道：App 内 + macOS 通知 + 声音
- 历史预警记录（可回看复盘）
- **减少用户盯盘时间，降低错过机会概率**

**⑦ 交易日志**（Stage A 新增 · 最高粘性功能）
- 半自动：交割单导入自动生成条目
- 手动补充：原因 / 情绪 / 计划偏差 / 教训
- 标签 + 搜索 + 月度总结
- **日积月累无法迁移 —— 强锁定效应**

**⑧ 模拟训练**（Stage A 新增 · 无实盘接入时的过渡形态）
- 接 CTP SimNow 仿真环境（零成本）
- 模拟盘 + 训练模式 + 纪律检查
- 计划 vs 实际偏差评分

**⑨ 工作区模板**（Stage A 新增 · Pro 工作流粘性）
- 多窗口布局保存与切换（如盘前/盘中/盘后三套）
- 快捷键一键切换 + CloudKit 云同步

**⑩ CTP 下单 · 订单体系**（Stage B 核心）
- Obj-C++ → Swift Package 封装
- **基础订单**（Stage B 初期必做）：
  - 市价单 / 限价单 / 止损单 / 止盈单 / 快捷键下单
- **高级订单**（Stage B 中期）：
  - 追踪止损（Trailing Stop）
  - OCO（One-Cancels-Other · 止盈+止损二选一，触发一单自动撤另一单）
  - 括号单（Bracket · 开仓+止损+止盈三单联动）
  - 冰山单（Iceberg · 大单拆小单，暴露量可控）
- **算法订单**（Stage B 末/C 期视客户需求）：
  - TWAP（时间加权平均）/ VWAP（成交量加权平均）
  - 阶梯单 / 狙击单

**⑪ 麦语言解析器**（战略武器）
- Stage A 晚期：基础版 30-50 函数，用户可运行公式
- Stage B 末：完整 95%+ 覆盖 + 策略执行引擎（**拆文华护城河的关键一刀**）

**⑫ iPad 专业工作流**（Stage B 独家）
- 6 大场景：出行盯盘 / 会议摆台 / Apple Pencil 分析 / Magic Keyboard 下单 / 外接 4K / Sidecar 协同
- **快期/文华因不在 Apple 生态几乎无法复制**

### 3.3 数据与分析体系（内嵌）

**产品事件埋点**（Stage A 必做）：
- 图表操作 / 指标添加 / 画线 / 订阅变更 / 崩溃
- Stage A（DAU ≤ 1000）：自建 SQLite + 手动 SQL 查询即可
- Stage B（DAU > 5000）：接入 Amplitude 或火山引擎

**A/B 测试**：feature flag 层已预留，Stage B 开始做 onboarding / 定价 A/B

**用户画像字段**：注册来源 / 交易品种偏好 / DAU / 功能偏好 / 最后活跃 / 续费预测分

### 3.4 性能基准（工程硬指标）

所有数字都是**红线**，超标的 PR 必须优化后再合入：

| 维度 | 目标 | 红线（触发回滚）| 说明 |
|------|-----|------------|------|
| **图表渲染** | **60 fps** | < 45 fps | Metal 自研引擎 |
| **10 万 K 线滚动** | 无掉帧 | 单帧 > 16ms | 核心差异化基线 |
| **冷启动** | **< 1s** | > 2s | 优先加载 UI + 懒加载数据 |
| **热启动** | < 500ms | > 1s | |
| **Tick 处理** | **< 1ms / tick** | > 5ms | 高频数据刚需 |
| **下单延迟**（Stage B）| **< 5ms**（本地 → CTP）| > 20ms | 专业用户容忍度 |
| **内存占用** | **< 500 MB**（1000 合约 + 6 图表）| > 1 GB | 避免拖累老设备 |
| **磁盘写入**（1 天行情）| < 100 MB | > 500 MB | 依赖压缩策略 |

**性能回归机制**：
- CI 跑 benchmark test（至少图表渲染 + 启动 + Tick 处理 3 项）
- 任何 PR 触发任一指标倒退 → **自动 block merge**
- 性能退化不是"后面优化" · 是**当下修或回滚**（产品原则 #3）

## 四 · 核心卖点

### 4.1 对外一句话

> **"为专业交易者打造的 Mac/iPad 原生专业工作台 —— 盯盘、预警、复盘、训练、日志，让你盯得更快、复得更清、练得更稳。"**

**定位关键**：**是"工作台"不是"交易终端"**。Stage A 主下单继续用现有软件，我们解决 Windows 老软件做得差的 5 件事。Stage B 加入 CTP 后成为完整终端。

### 4.2 差异化 6 维（必须同时成立）

1. 审美与性能（SwiftUI + Metal 60fps）
2. Mac 原生（文华没有，快期平庸）
3. Apple 生态整合（Widgets / Shortcuts / Focus）
4. Mac + iPad + iPhone 协同（单账户三端同步）
5. 透明定价 + 不卖数据
6. **麦语言完整兼容**（拆除文华最大护城河）

### 4.4 话术库

## 十二 · 深化补充（6 个战略议题）

### 12.1 品牌调性体系（非 IP 品牌人格）

你选了"创始人 IP 不重要"，品牌需自成人格：

**品牌人格 5 词**：**克制 · 专业 · 挑剔 · 冷静 · 有品**

**视觉基调**：
- 主色：深灰 + 单一亮色（**非红绿**，避免交易红绿混淆）
- 字体：苹方 + SF Mono（代码感）
- 界面密度：中偏高（Pro 用户能容纳信息）

**内容语调**：
- 不煽情 / 不卖惨 / 不诉诸焦虑
- 用数据说话（性能/覆盖率/延迟）
- 拒绝"AI 赋能""颠覆式""重新定义"

### 12.2 用户信任曲线（金融工具特殊性）

**18 个月信任累积**：

```
M0 ─ M3 ─ M6 ─ M9 ─ M12 ─ M18 ─ M24
下载  体验  付费  续费  口碑    推荐    粉丝
      │    │    │    │      │
      │    ●    │    │      │
      │    首次 │    │      │
      │    错单 │    │      │
      │    毁12月信任       │
              │              │
              零事故累积 ───→ 朋友间推荐
```

**关键认知**：
- 第一次错单 = **毁 12 个月信任**（需 6 月修复）
- 18 月零事故 = 能推荐给朋友
- 24 月零事故 = 品牌资产真正成立

**行动指南**：稳定性 > 创新（产品原则 #7 的数据化体现）

### 12.3 竞争反应预案

**场景 1 · 文华推 Mac 精品版**（概率低 / 影响高）
- 信号：发布会 / Swift 工程师招聘
- 对策：加速 iPad 专业工作流 + 麦语言覆盖率到 95%

**场景 2 · 快期推 Mac 精品版**（概率中 / 影响中）
- 信号：交易所合作公告
- 对策：品牌区隔（我们 Pro / 快期免费）+ iPad 先发护城河

**场景 3 · 新玩家进场**（概率高 / 影响低）
- 对策：Stage A 末先发口碑即可

### 12.4 国际化预埋（Stage C 考虑）

**当下零成本预埋**：
- 字符串资源化（为 i18n 留接口）
- UTF-8 + 时区无关
- 支付层解耦（未来接 Stripe）

**可及市场**：
- 海外华人交易者（港新马北美）：10-30 万
- CME / SGX / HKEX 合约扩展

**触发条件**：国内 ARR ≥ ¥2000 万 + 明确海外需求信号

### 12.5 数据资产价值（合规使用）

**价值分层**：
- 聚合匿名数据（总用户数、热门品种）→ 可公开
- 用户行为画像（脱敏）→ 可商业化但不卖
- 交易策略原文 → 用户产权，不碰

**合规商业化路径**：
- ❌ 不卖数据
- ✅ 出**行业报告**（品种热度趋势、用户分布）免费公开 → 品牌权威
- ✅ 与券商合作**聚合数据洞察**（不涉个人）→ 换合作资源

### 12.6 退出路径设计（3-5 年视角）

| 路径 | 概率 | 估值区间 | 时间 |
|------|-----|--------|------|
| 被金融科技公司并购（同花顺/恒生/券商）| **60%** | ¥5-20 亿 | M36-M60 |
| 被期货行业头部并购 | 20% | ¥3-10 亿 | M36-M60 |
| 独立盈利发展（ARR ¥1-3 亿 保持独立）| 15% | — | M36+ |
| 独立上市 | 5% | ¥10+ 亿 | M60-M84 |

**预埋动作**：
- Stage A 搭建 **开曼 + BVI + WFOE + 境内运营** 四层结构（律师辅助）
- 数据 / 商标 / 代码版权清晰归属
- 合伙人股权回购条款写清
