## 5 · 关键设计沉淀（v2.0 → v3.0 新增）

### Sources/Shared 跨端层模式
- **协议优先**（KLineCacheStore / AlertHistoryStore / JournalStore / FeatureFlagStore）+ **多实现并存**（InMemory + 文件 / SQLite v2 / CloudKit v2）
- **CloudKit 字段映射预埋**（Watchlist + Workspace + 后续 Journal）：不 import CloudKit，仅字段契约预埋
- **时间外置**：所有 actor 不持 Timer/Task，时间通过 `now: Date` 注入参数 → 测试 100% 确定性
- **AsyncStream 多订阅者模式**（ConnectionStateMachine / SimulatedMarketDataProvider / UnifiedDataSource / ReplayPlayer / AlertEvaluator）：UUID 字典管理 + onTermination weak self
- **broadcast helper DRY**（ReplayPlayer / AlertEvaluator）

### 业务规则沉淀
- **多合约严格隔离**（push 按 instrumentID 精确分发）
- **频控冷却**（A08 验收硬要求；canTrigger + cooldownSeconds + lastTriggeredAt 三字段）
- **单向引用**（A09 禁做：日志 tradeIDs 不污染 trades）
- **显式转换边界**（A09 禁做：RawDeal.toTrade() 唯一通道）
- **统一通知层**（A08 禁做：NotificationChannel + Dispatcher）
- **业务唯一入口**（D2 §2：FeatureFlagService.isEnabled 不散落 if flag.xxx）
- **FIFO 配对算法**（多空方向自动推导 + 4 种平仓 flag 全识别 + 部分平仓拆分 + 手续费比例分摊）

### Karpathy 4 要点持续落实
- code-simplifier 14 轮过审 · **累计净 -22 行**（DRY + 显性化 + 防御性）
- 每个 WP 完成后立即 simplifier + git diff（已成定式）

### 三方向候选

| 方向 | 候选 WP | 工作量 | 战略价值 | 阻塞 |
|------|--------|--------|---------|------|
| **🅰 切 Mac 启动核心差异化** | WP-20 Metal K 线 PoC（前哨）→ WP-40 Metal 引擎（6-8 周） | 1-2 周 → 6-8 周 | ⭐⭐⭐⭐⭐ M6 生死核心 | 需 Mac 持续时间 |

| **🅲 清非工程 M1 必前置** | WP-10 股权协议 / WP-02 律师咨询 / WP-100 品牌命名 | 1-3 天 / 1 次 / 3-5 天 | ⭐⭐⭐⭐ 法律 + 品牌底线 | 你 / 律师 / 顾问 |

### 我的最佳推荐

🥇 **🅱 + 🅲 并行**：

**🅱 主线**：切 Mac 上 WP-21b（3-5 天）—— 真行情接通 → 后续所有数据层 WP 立即跑真实数据回归（指标 / 画线 / 自选 / 缓存 / 回放 / 预警 / 复盘 / 日志）。性价比最高的 Mac 工作。

**🅲 副线**：你**今天就**联系律师约 WP-02 时间（1-2 周后回访）+ 起草 WP-10 股权协议初稿（合伙人 Hunter 之前没跟你正式签 → 必须落实）。这两件不需要工程时间。

**为什么不推 🅰 Metal**：6-8 周大块工作 + Mac 切机持续时间长，应等 21b 真行情接通后再启动（让 Metal demo 有真数据可演）。

**为什么不推 🅴 用户访谈**：你已经是 Pro 交易者本身，自己就是 Persona 1。M1 访谈优先级降到与 21b 同期。

### 切换条件

- 想**纯做产品体验闭环** → 🅰 WP-20 Metal PoC（直接看 K 线渲染）
- 想**先填合规底线** → 🅲 WP-02 律师 + WP-84 合规设计
- 想**做后端选型** → 🅳 E8 基础设施（需选定 CloudKit vs 阿里云 + Go vs Swift on Server）
- 想**继续 Linux 工作流** → 没有合适的工程 WP，可考虑会话交接文档主文档 §19/§20 同步 + 死亡率自检报告（WP-90 系列）
