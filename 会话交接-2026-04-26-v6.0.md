### 5.1 WP-44c · MarketDataProvider 同合约多 handler 字典（commit 18eb68e · 来自端到端 demo 暴露的真问题）

#### 来源

v5.0+ 第 7 个 demo（EndToEndDemo）暴露：同合约 RB0 不能同时被 UDS（K 线合成）+ AlertEvaluator（预警 onTick）订阅，因为 `MarketDataProvider.subscribe` 是 `handlers[id] = handler` 字典覆盖语义。

#### 核心改动（4 文件协同）

- **协议 `MarketDataProvider`**：
  - `typealias SubscriptionToken = UUID`
  - `subscribe(_:handler:) async -> SubscriptionToken`（@discardableResult）
  - `unsubscribe(_:token:) async`（精确退订）
  - `unsubscribe(_:) async` 保留（清空合约所有 handler · disconnect 走此路径）
- **3 个 provider 实现统一改 `[String: [SubscriptionToken: handler]]`**：
  - `SinaMarketDataProvider`（pollOnce 内 `for handler in bucket.values { handler(tick) }`）
  - `SimulatedMarketDataProvider`（push 同 dispatch）
  - `MockMarketDataProvider`（Tests Helper · 行为与生产一致）
- **`UnifiedDataSource`**：保存 `realtimeTokens [String: SubscriptionToken]` · cleanup 用 token 精确退订
- **新增内省 API**：`handlerCount(for:)` 同合约多订阅者计数

#### 受益验证（EndToEndDemo v2）

- 段 3 改 RB0+IF0（原 AU0+IF0）
- RB0 同时被 UDS + AlertEvaluator 订阅
- 运行时 `handlerCount(for: "RB0") == 2` + 单次 HTTP 拉取 bucket 内 2 handler 都收 tick
- RB0 必触发预警 ×2 + IF0 不触发 ×0 + 段 2 UDS 仍合成 K 线

#### 测试

+6 测试 +1 suite（WP-44c 多 handler）：sameInstrument / byToken / lastToken / unsubscribeAll / mixed / unknownToken

## 6 · 关键设计沉淀（跨 WP 共通模式 · v6.0 增量）

### 6.1 Sources/Shared 跨端层模式（v6.0 增量）

- **协议优先 + 多实现并存**（WatchlistBookStore / WorkspaceBookStore 双 InMemory + SQLite）
- **SQLCipher 加密直通**：所有 SQLite store 加 `init(path:passphrase:)` 重载 · passphrase=nil 时行为同原生 SQLite
- **整本 Book 单表 JSON UPSERT 模式**（WatchlistBook / WorkspaceBook · id=1 单例 + CHECK 约束）
- **脏 JSON 显式抛 decodeFailed**（不静默吞数据 · UI 层须感知）

### 6.2 actor + AsyncStream 模式（v6.0 增量）

- **token 化精确退订**（MarketDataProvider · WP-44c）：subscribe 返回 SubscriptionToken · unsubscribe(_:token:) 不影响同合约其他 handler
- **actor 包 Task 自动驱动循环**（ReplayDriver）：start 启 Task.detached 循环 · 每步动态读状态 · 多重自动停条件
- **inline 错误处理**（SQLiteConnection 加密路径）：actor isolation 限制 nested func 不能改 self · 必须 inline

### 6.3 业务规则沉淀（v6.0 增量）

- **历史 + 实时合并去重**（UDS v2）：cache 优先（KLineBuilder 严格合成）· fetch 失败静默回退 · 仅支持 minute5/minute15/hour1
- **跨 actor 静态 formatter 复用**（ConsoleChannel / FileChannel / 多 demo）：`static let priceFormatter` 避免每次 send/print alloc
- **协议向后兼容扩展**（NotificationChannelKind 加 console + file · MarketDataProvider 加 token 重载）：rawValue 与 case 名对齐 · @discardableResult 让老 caller 不破

### 6.4 Karpathy 4 要点持续落实（v6.0 增量）

- code-simplifier **31+ 轮过审**（v5.0 18 + v6.0 13）
- 反复出现的优化模式：
  - `(try? Init) == nil` 替代 do/catch + var 标志
  - `static let DateFormatter / NumberFormatter` 抽常量（避免每次 alloc）
  - 数组 + for 循环替代 reduce / 链式 +
  - guard let + 早返回扁平化
  - 命名常量替代 magic number（frameBudget60fps / healthyFrameTolerance）
  - actor 不能继承 / cross-actor 重复代码"无可改"明说

### 6.5 v6.0 流程修订：simplifier 在 build/test 之前

**用户明确纠正**（feedback_编码规则.md 已更新）：
- ❌ 错误顺序：写代码 → build → test → simplifier → 再 build/test → commit
- ✅ 正确顺序：写代码 → **simplifier** → build/test/run（一次到位） → commit

理由：
- 简化后的代码才是要测试的代码（不要测旧版本然后再改）
- build/test/run 跑两遍浪费时间（简化前 + 简化后各一遍）
- 一次走通流程更紧凑

## 11 · 可能踩的坑（v6.0 增量）

> v5.0 已记录 19 个坑保留有效，本节仅列 v6.0 新增。

| # | 现象 | 原因 | 解决 |
|---|------|------|------|
| 20 | **SQLCipher header 找不到 sqlite3_key 符号** | sqlcipher/sqlite3.h 的 sqlite3_key 在 `#ifdef SQLITE_HAS_CODEC` 包围下；client 端必须定义此宏 | shim.h 加 `#define SQLITE_HAS_CODEC 1` 后再 `#include <sqlcipher/sqlite3.h>` |
| 21 | **Swift 6 actor init 内 nested func 不能改 self.db** | 严格并发 actor-isolated property 不能从 nonisolated context 修改；nested func 默认 nonisolated | 改 inline · 不抽 helper · 重复 3-4 行接受 |
| 22 | **Swift 6 跨 main actor 边界传非 Sendable 闭包** | `timed<T>(_ work: () async throws -> T)` 闭包默认非 Sendable | helper 加 `T: Sendable` + `@Sendable () async throws -> T` 闭包标注 |
| 23 | **AnalyticsEvent.init 参数 name → eventName + 类型 String → AnalyticsEventName 枚举** | API 演进 · 测试代码用旧签名 | demo 用 `.appLaunch` 等 enum case 替代 String |
| 24 | **三层 Optional 表达式编译挂掉** | `(try? MA.calculate(...))?[0].values.last` 三层 Optional · Swift 类型推断挂 | 拆 `guard let result = try? ...` + `result.first?.values.last ?? nil` |
| 25 | **String.padding 与 padded 自定义 extension** | EndToEndDemo / JournalGeneratorDemo / WenhuaCSVImportDemo 等多 demo 用 `private extension String { padded(_:) }` | 各文件 `private` 限定 · 不冲突 |
| 26 | **ContractStore productID 大小写不一致** | `byProduct(productID.uppercased())` 但 upsert 直接用 contract.productID 不 uppercase | 项目惯例 productID 用大写（"RB"/"IF"/"AU"）· 测试 sampleJSON 也用大写 |
| 27 | **ContractStore 是 final class 不是 actor** | `await store.byProduct(...)` 触发 "no 'async' operations" warning | 去掉所有 `await store.` 前缀 |
| 28 | **AsyncStream 多 for-await 顺序消费** | UDSHistoryMergeDemo 段 1 firstSnapshot break 后段 3 再 for-await 同一 stream | 实测 OK（AsyncStream buffer 持续累积）· simplifier 提示但不阻塞 |
