## 6 · 关键设计沉淀（v7.0 增量）

### 6.1 actor → final class @unchecked Sendable + NSLock（替代 actor 模型）

**根因**：Swift 6 严格 concurrency · MTLRenderPassDescriptor / MTLDrawable 非 Sendable · 跨 actor 边界要 sending 修饰符 · 引发连锁报错（sending result task-isolated · 无解死循环）。

**方案**：
- `public final class MetalKLineRenderer: KLineRenderer, @unchecked Sendable`
- 内部 NSLock 序列化可变状态
- 协议 async 方法签名包装 + sync 私有 helper（NSLock 不能在 async context 调用 · 防止跨 await 死锁）
- 渲染同步执行 · 不开 Task · MTKViewDelegate.draw 同步调用

**适用范围**：所有 Metal 渲染层 actor 替换 · M5 Mac App 集成同模式。

### 6.2 KLineVertex 6 Float packed（替代 SIMD2+SIMD4）

**根因**：SIMD4<Float> alignment=16 让 struct stride 自动 pad 24 → 32 · vertex descriptor offset 错位 · GPU 读到垃圾 · 颜色全错。

**方案**：6 个独立 Float 字段（x/y/r/g/b/a · align 4 · 无 padding · stride 严格 24）· init 接 SIMD2/SIMD4 透明转换。

**经验**：vertex 结构必须 packed Float · 不用 SIMD（SIMD 仅用于 viewMatrix 等计算用途）。

### 6.3 sub-bar 浮点偏移（pixel-precise pan）

**问题**：startIndex Int · pan 必然阶梯跳跃（visible=200 时拖 1 px ≈ 0.16 K → Int 取整 0 或 1 · 体感"咔哒咔哒"）。

**方案**：协议加 `startOffset: Float (0..<1)` · pannedSmooth(byBars: Float) 浮点平移 · 自动累加 + 满 1 进位 · viewMatrix xLeft 用 startIndex + startOffset。

**意义**：所有图表 pan 交互的标配设计 · WP-40 完整版 / WP-43 自选切换 / WP-44 多窗口都复用。

### 6.4 惯性滚动（Task @MainActor + 衰减）

**模型**：
- onEnded 用 SwiftUI DragGesture.predictedEndTranslation 算系统预估剩余位移
- 分摊到 inertiaSpreadFrames=20 帧得初始速度
- 启动 Task @MainActor · 60fps loop（16.67ms sleep · ProMotion 120Hz 自动叠加）
- 每帧 viewport.pannedSmooth(byBars: v) · v *= 0.97 衰减
- 触底（clamp 让 viewport 不变）立即停止 · 用户重新 drag/zoom 取消

**调参**：衰减率 0.97 ≈ 2 秒衰减 · 阈值 0.02 让轻甩也触发 · 分摊 20 帧让初速度足够大。

### 6.5 NSHostingController + setContentSize（替代 NSHostingView）

**问题**：`window.contentView = NSHostingView(rootView:)` 不自动 stretch · NSHostingController.intrinsicContentSize 取 SwiftUI ContentView idealSize（HUD 200x100）· 窗口缩成指甲盖。

**方案**：
- `window.contentViewController = NSHostingController(rootView: rootView)`（自动 layout）
- `window.setContentSize(NSSize(1280, 720))` 强制覆盖
- ContentView body 加 `.frame(minWidth: 800, idealWidth: 1280, minHeight: 480, idealHeight: 720)`
- MTKView 内加 `autoresizingMask = [.width, .height]`（AppKit 层 fallback）

### 6.6 Metal 渲染双路径（UI vs benchmark）

**模型**：
- UI 路径（renderToDrawable）：fire-and-forget · 不 waitUntilCompleted · 主线程立即返回 · 拖拽丝滑
- benchmark 路径（renderHeadless）：waitForCompletion=true · 精确测 GPU 完成时间
- private encodeAndCommit 接 waitForCompletion: Bool 参数

**适用**：所有渲染器都应分两路径 · 生产代码 UI 异步 · 性能验证同步。

### 6.7 Indicator 同值像素覆盖（BOLL-MID = MA(20) 坑）

**现象**：BOLL-MID 与 MA(20) values 完全相同 · 后绘制覆盖前绘制 · 视觉只见 N-1 条线。

**方案**：demo 端过滤同值 indicator（demo 已知 MA(20) 与 BOLL period=20 重合 · 仅保留 UPPER/LOWER）。

**长期**：IndicatorSeries 加显式 color 字段 / 或者 renderer 检测 hash 重复跳过。

### 6.8 HUD 数值取 visible window 末位

**问题**：indicator 全段（如 100k 长度）末位与 visible window 末位（如 startIndex+visibleCount-1）不同 · random walk 漂移让数字差很多。

**方案**：`series.values.prefix(viewport.startIndex + viewport.visibleCount).compactMap.last`。

## 11 · 可能踩的坑（v7.0 新增 10 个 · v6.1 33 个保留）

| # | 现象 | 原因 | 解决 |
|---|------|------|------|
| 34 | **Swift 6 sending 修饰符跨 actor 死循环** | actor 接收 MTLRenderPassDescriptor 等非 Sendable Metal 类型 · 要 sending · sending result task-isolated 无解 | actor → final class @unchecked Sendable + NSLock 序列化（彻底放弃 actor 模型）|
| 35 | **NSLock.lock/unlock 在 async context 被禁** | Swift 6 strict mode 防 await 跨锁死锁 | async 方法仅作签名包装 · NSLock 操作下沉到 sync 私有 helper |
| 36 | **SIMD4<Float> alignment=16 让 struct 自动 pad** | struct 含 SIMD4 时编译器强制对齐 16 · stride 24 → 32 · vertex descriptor offset 错位 GPU 读垃圾（颜色全错） | KLineVertex 改 6 Float packed（x/y/r/g/b/a · align 4 · 无 padding · stride 严格 24）|
| 37 | **NSHostingView 不自动 stretch SwiftUI** | NSHostingController.intrinsicContentSize 取 ContentView idealSize · 窗口缩成指甲盖 | NSHostingController + setContentSize · ContentView .frame(idealWidth/idealHeight) · MTKView autoresizingMask |
| 38 | **NSViewRepresentable 在 ZStack 默认 zero size** | SwiftUI ZStack 不强制 stretch children · MTKView drawableSize=0 · currentDrawable nil · 不渲染（白底） | 加 .frame(maxWidth: .infinity, maxHeight: .infinity) |
| 39 | **waitUntilCompleted 阻塞主线程 ~1.5ms/帧** | 为精确测 stats 同步等 GPU · 60fps drag 期间累积成卡顿 | UI 路径 fire-and-forget（renderToDrawable）· headless 路径保留 wait（renderHeadless）· encodeAndCommit 加 waitForCompletion 参数 |
| 40 | **MTKView preferredFramesPerSecond=60 限制 ProMotion** | M4 Pro 屏 120Hz · 设 60 反让 drag 视觉只屏幕一半更新频率 | 改 120 · 系统取屏幕实际上限（普通 60Hz 屏自动降回） |
| 41 | **DragGesture 默认 minimumDistance=10 · 起手延迟** | trackpad 拖 10 px 才响应 · 体感"先滑一段才动" | DragGesture(minimumDistance: 0) |
| 42 | **startIndex Int 阶梯跳跃 · 力度与位移不成正比** | pan 必然按整数 K 跳 · visible=5000 时拖 5 px 仅跳 1 K · 实际应跳 ~20 K | 协议加 startOffset: Float · pannedSmooth(byBars: Float) · viewMatrix xLeft 用 startIndex+startOffset |
| 43 | **拖拽缺惯性 · 一松手就停（老代码体验是轻甩飞远）** | DragGesture.onEnded 默认无动量 | onEnded 用 predictedEndTranslation 算速度 · 启动 Task @MainActor 衰减 loop（0.97 衰减 · 0.02 阈值 · 20 帧分摊） |
| 44 | **BOLL-MID = MA(period) 像素级覆盖** | BOLL 公式 MID = MA(close, N) · 与同 period MA 完全同值 · 后绘制覆盖前绘制 · 视觉缺一根 | demo 端过滤 BOLL-MID（用 MA20 顶替 · 文华专业版同款）· 或将 indicator name 加显式 color 字段 |
| 45 | **HUD indicator 末值取全段不取 visible 末位** | indicator 100k 长度 · 末位与用户视野末位（startIndex+visibleCount-1）不同 · random walk 漂移让数字差很多 | latestText 改 series.values.prefix(visibleEnd).compactMap.last |
