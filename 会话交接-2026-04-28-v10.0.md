### 5.2 WP-41 v3 IndicatorCore 增量大扩展（9 批 17 commit · 5 → 25 指标）

#### v3 第 1 批 commit 1-4（KDJ + CCI + ATR + benchmark 8 指标 · 4 commit）

- **KDJ**（`9cebc3a`）：内嵌 high/low ring buffer · 每步 O(n) 扫描 max/min（n=9 微秒级 · 简单优于 monotonic deque）· prevK/prevD 流式 SMA 平滑（中国文华风格）· J = 3K - 2D · 与 calculate hhv==llv → rsv=0 一致
- **CCI**（`64a1f21`）：TP=(H+L+C)/3 滑窗 sum + O(n) MD 重算（不预存 |TP-MA| 累加 · 增量更新代价不省）· factor=0.015
- **ATR**（`30bacd9`）：Wilder O(1) per step · processStep 4 阶段（第 1 根 TR=high-low / 第 2..n-1 累加 / 第 n 根 seed atr=warmUpSum/n / 第 n+1 起 Wilder 平滑）
- **benchmark + 收官**（`f87d062`）：扩展到 8 指标 · Mac 验收 +25 项

#### v3 第 2 批 commit 1-4（OBV + WR + ADX + DMI + benchmark 12 指标 · 4 commit）

- **OBV**（`c3259de`）：累积式无周期无 warm-up · 第 1 根 = volume · 第 2 根起按 close 涨跌累加/扣减/不变（与 calculate 一致）
- **WilliamsR**（`4ba5501`）：KDJ ring 简化版（state 4 字段 = KDJ 13 字段 1/3 · 无 SMA 平滑）· hhv == llv → nil
- **ADX + DMI**（`f029aa3`）：4 路 Wilder（tr/plusDM/minusDM 一级 + dx → ADX 二级）· 关键精度对齐：用 round8(state.atr/smPDM/smMDM) snapshot 算 +DI/-DI（同 RSI commit 2/4 修正）· DMI typealias = ADX.IncrementalState（零开销复用）
- **benchmark + 收官**（`5716e98`）：扩展到 12 指标 · Mac 验收 +25 项

#### v3 第 3 批（Stochastic · 1 commit）

- **Stochastic**（`f3963be`）：双 ring HHV/LLV（n=14）+ %K_raw ring（s=3）+ kSum 滑动给 %D · 每步 %K_raw 必写 ring（含 0 · 与 calculate kRaw 数组同语义）· 全平时 %K nil + %D 始终 0
- benchmark 13 指标 · Mac 验收 +8 项

#### v3 第 4 批（TRIX · 1 commit）

- **TRIX**（`b8b7d5b`）：内嵌 3 EMA.IncrementalState + prevE3 round8 差分（与 calculate e3 数组中 round8 值一致）· 与 MACD 区别：TRIX e2/e3 每步无条件 advance（用 e1/e2 ?? 0 · 与 Kernels.nextEMA 行为一致）· MACD signal 仅在 fv/sv 都有效时 advance
- benchmark 14 指标 · Mac 验收 +8 项

#### v3 第 5 批（DEMA + TEMA · 1 commit）

- **DEMA**（`1a7bdc3`）：内嵌 2 EMA · 输出 2*e1 - e2（同 TRIX 复合 EMA 模式简化 · 无差分）
- **TEMA**：内嵌 3 EMA · 输出 3*e1 - 3*e2 + e3
- benchmark 16 指标 · Mac 验收 +10 项

#### v3 第 6 批（VWAP + PSY + CMO · 1 commit）

- **VWAP**（`3c56c21`）：累积 typical*volume / 累积 volume（同 OBV 模式无周期 · 无 warm-up · cumV>0 输出）
- **PSY**：单 ring sliding sum 上涨标志 0/1 · count == period 起输出
- **CMO**：双 ring up/dn sliding sum（同 RSI 思路无 Wilder）· count > period 起输出（跳过 i=n-1 与 calculate 一致）
- 加速亮点：CMO 1.8×（消除 calculate 中 slidingSum 双数组分配）
- benchmark 19 指标 · Mac 验收 +12 项

#### v3 第 7 批（ROC + BIAS · 1 commit）

- **ROC**（`be47747`）：最简 ring · 先取 ring[head]（即将被覆盖的最旧 = close[i-n]）再写入 · countBefore == period 守卫等价 calculate `for i in n..<count` 第一根 i=n
- **BIAS**：ring SMA + (close - ma) / ma * 100 · 关键修正：raw ma → round8(ma) snapshot（实测末位差 1 · 同 RSI/ADX 同模式精度对齐）
- benchmark 21 指标 · Mac 验收 +10 项

#### v3 第 8 批（WMA + HMA · 1 commit · 最高加速）

- **WMA**（`3776f44`）：**O(1) Pascal triangle sliding** 核心优化：
  - 公式：`num[i+1] = num[i] + n*close[i+1] - runningSum[i]` / `runningSum[i+1] = runningSum[i] + close[i+1] - close[i-n+1]`
  - warm-up 期 count==period 时一次性 seed numerator + runningSum（O(n) · 仅一次）
  - sliding 期每步 O(1)（Decimal 加减乘精确 · 与 calculate 每步 O(n) 重新累加数学等价）
  - WMA.IncrementalState.advance(close:) public mutating method（同 EMA.advance 模式 · 给 HMA 内嵌用）
  - **benchmark 2.3× 第 8 批最高加速**（仅次于 MA 2.5×）
- **HMA**：内嵌 3 WMA（halfN=8 / n=16 / sqrtN=4）· raw = 2*wHalf - wFull · final = wma(raw, sqrtN) · 中间层 hmaWma 每步无条件 advance · 受益 3 路 WMA O(1) Pascal sliding · benchmark 1.7×
- benchmark 23 指标 · Mac 验收 +12 项

#### v3 第 9 批（PVT + Donchian · 1 commit · 跨 Volume + Volatility 类）

- **PVT**（`239c1f6`）：量价类累积式（同 OBV 模式 · 无周期）· 流式累加 (close-prev)/prev * volume · 第 1 根 PVT=0（无 warm-up）
- **Donchian**：波动率类双 ring HHV/LLV（同 KDJ ring 模式）· 输出 [upper, mid, lower] 3 列 · upper/lower 是 raw HHV/LLV（不 round8 · 与 Kernels.hhv/llv 一致）· mid = round8((upper+lower)/2)
- benchmark 25 指标 · Mac 验收 +14 项

## 6 · 关键设计沉淀（v10.0 增量 · 跨 commit 共通）

### 6.1 IncrementalIndicator 协议生态扩张（5 → 25 指标 · 9 批模式）

按算法模式分类：
1. **环形 buffer + 滑动 sum**（O(1)）：MA / BIAS · 同 SMA 框架
2. **环形 buffer + O(n) 扫描**（n 小可接受）：KDJ / WR / Stochastic / Donchian · HHV/LLV 类
3. **流式 prev EMA**（α 平滑）：EMA / TRIX / DEMA / TEMA · advance(close:) 共享接口
4. **流式 prev Wilder**（n=1/n 平滑）：RSI / ATR / ADX / DMI · processStep 4 阶段
5. **复合内嵌 EMA**（同 MACD 模式）：MACD / TRIX / DEMA / TEMA / HMA（内嵌 WMA）
6. **累积式无周期**（无 warm-up）：OBV / VWAP / PVT
7. **滑动 sum 差分**（双 ring）：CMO（同 RSI 思路无 Wilder）
8. **O(1) Pascal triangle sliding**（线性加权 sliding 优化）：WMA · `num[i+1] = num[i] + n*close[i+1] - runningSum[i]`
9. **TP+MD 双层**（O(n) MD）：CCI · TP=(H+L+C)/3 滑窗 sum + |TP-MA| 重算

### 6.2 round8 snapshot 精度对齐模式（关键陷阱）

**症状**：增量与全量 calculate 末位差 1-2 位（如 41.59471751 vs 41.59471748）

**根因**：calculate 中用 `Kernels.ema/wilder/ma` 输出的 round8 数组作下游计算输入；增量 state 内 prev 是流式 raw（未 round），直接做下游差末位精度。

**修正模板**（v3 多次重复）：
```swift
let smap = Kernels.round8(state.prev)   // 取 round8 snapshot
guard smap > 0 else { return nil }
return Kernels.round8(... formula with smap ...)
```

**已应用**：RSI commit 2/4（首次发现）· ADX commit 3（4 路 Wilder + DI 计算）· BIAS commit（ring SMA）

### 6.3 WMA.IncrementalState.advance(close:) public 接口模式

**目的**：HMA 等复合指标内嵌 WMA · 不需通过 stepIncremental(state:newBar:)（构造 fake KLine）。

**模式**：
```swift
public struct IncrementalState: Sendable {
    public mutating func advance(close: Decimal) -> Decimal? { ... }
}
public static func stepIncremental(state: inout IncrementalState, newBar: KLine) -> [Decimal?] {
    [state.advance(close: newBar.close)]
}
```

**已应用**：EMA（v2 commit 2/4）· WMA（v3 第 8 批） · 后续 HMA 内嵌 3 WMA · MACD/TRIX/DEMA/TEMA 内嵌 EMA。

### 6.4 ADX state typealias 零开销复用（DMI = ADX 子集）

```swift
extension DMI: IncrementalIndicator {
    public typealias IncrementalState = ADX.IncrementalState
    public static func makeIncrementalState(...) throws -> IncrementalState {
        try ADX.makeIncrementalState(...)
    }
    public static func stepIncremental(state: inout IncrementalState, newBar: KLine) -> [Decimal?] {
        let row = ADX.stepIncremental(state: &state, newBar: newBar)
        return [row[1], row[2]]   // [+DI, -DI]
    }
}
```

benchmark 验证：DMI 与 ADX 性能几乎一致（typealias + state 共享 · 零额外开销）。

## 11 · 可能踩的坑（v10.0 新增 9 个 · v9.0 之前坑保留）

1. **WMA Pascal triangle 顺序**：先用旧 runningSum 算 numerator · 再更新 runningSum · 顺序反了 numerator 用错
2. **WMA seed 时机**：count < period 时 ring 累加 + 不输出；count == period 时一次性 seed numerator + runningSum（O(n) · 仅一次）· 不能在每次写 ring 时维护 numerator（warm-up 公式不同）
3. **HMA 内嵌 WMA halfN/n/sqrtN**：不是同一 period · 三个独立 WMA.IncrementalState 各自 advance · raw 用 round8(wHalf)/round8(wFull)
4. **TRIX vs MACD signal EMA advance 行为不同**：TRIX e2/e3 每步无条件 advance（用 e1/e2 ?? 0 · 与 nextEMA 一致）· MACD signal 仅在 fv/sv 都有效时 advance（calculate firstDIFIdx 切片）· 误用会导致计数错乱
5. **CMO 第一根输出在 count==period+1**：calculate `for i in n..<count` 跳过 slidingSum 第一个非 nil（i=n-1）· 增量用 count > period 守卫等价（不是 count == period）
6. **ROC 取 ring[head] 时机**：先取（即将被覆盖的最旧 = close[i-n]）· 再写入新值 · 反了取到当前 close
7. **BIAS round8 snapshot**：calculate Kernels.ma 输出是 round8 数组 · 增量必须用 round8(ma) 作下游 (close-ma)/ma · 否则末位差 1（同 RSI/ADX 同模式）
8. **PVT 第 1 根 = 0**：与 calculate out[0] = 0 一致 · prevClose=nil 时 acc 不变 · 输出 round8(0) = 0 · 不能跳过第 1 根
9. **Donchian upper/lower 不 round8**：calculate hhv/llv 输出 raw（max/min 无精度损失）· 增量也直接 return raw · mid = round8((upper+lower)/2)
10. **Stochastic %K_raw 每步必写 ring**：含 0 写入（与 calculate kRaw 数组每位都赋值一致）· kSum 滑动 sum 需要历史 0 值参与
11. **Donchian / Stochastic 双 ring 同步推进**：highRing 和 lowRing 必须每步同时写 · 否则 hhv/llv 计算错位
12. **ADX 4 路 Wilder 精度对齐**：用 round8(state.atr/smPDM/smMDM) snapshot 算 +DI/-DI · 实测首次实现差 0.00000003-0.00000010 末位
13. **WP-55 跨 Kind 拖拽**：v1.5 用 WorkspaceTemplateRef.kind 守卫拒绝 · 不允许跨 Kind 移动（→ contextMenu「修改类型」）· UI 简化语义清晰
14. **WP-55 v1.7 JSONEncoder.cloudKit 是 internal**：Workspaces module 内 internal · 不跨模块 · MainApp 必须自己定义 static encoder/decoder（注释说明）
15. **WP-55 v2 mini preview frame 全 zero**：未应用网格的 windows · rect 都重叠 (0,0) · 视觉提示"未配置布局"（不是 bug）
