### 5.2 v3 系列收官里程碑

**41/44 = 93.2% 协议允许范围全覆盖**

剩余 3 个不可严格增量化的指标：
- **SAR**：calculate out[0] 依赖 close[1] 决定方向 · 用未来信息（增量协议不支持）
- **Fractal**：需要 i+2 才能判定 i 中心点 · 用未来信息（5 根 lookback 中心判定）
- **ZigZag**：非线性 · 标记 pivot 时回写历史（增量无法回写）

**v3 系列模式沉淀（16 批积累 · 12 大可复用模式）**：
1. 滑动 sum 类（O(1)）：MA / BIAS / CMF（双 sliding sum）/ VR（三桶 sliding sum）
2. HHV/LLV ring 类：Donchian / KDJ / WR / Stochastic / PriceChannel
3. prev EMA Wilder 类：EMA / RSI / ATR / ADX / TRIX
4. 复合 EMA 类：MACD / TRIX / DEMA / TEMA / HMA（内嵌 WMA） / KC / VOSC / Supertrend

### 6.3 isLoaded 守卫（v11.0 关键防误写设计）

**问题**：SwiftUI `.onChange` 在 Mock 默认值首次进入 view tree 时会立刻被 trigger（即使 Mock 进入也算 change）· 不加守卫会出现：
- 启动 → init State Mock → 渲染 → onChange Mock save → 覆盖磁盘真数据

**解决**：`@State isLoaded: Bool = false` 守卫 · `.task` load 完成后才 isLoaded=true · onChange 内 `guard isLoaded` 跳过 Mock 阶段。

**例外**：ChartScene 不需要 isLoaded（pipeline 网络数据本身就是真数据 · 不会"误写"）。

### 6.4 协议形态差异下的 Optional 解构（4 种模式）

| 协议返回 | try? 后类型 | 解构写法 |
|---|---|---|
| `T?` 单层 | `T??` | `let outer = try?, let loaded = outer { ... }` 或 `(try?) ?? nil` |
| `T` 非 Optional | `T?` | `if let loaded = try? { ... }` |
| `[T]` 数组 | `[T]?` | `let arr = (try?) ?? []` + `!arr.isEmpty` 判空 |
| `Bool` 等值 | `Bool?` | `if (try?) == true` |

### 6.5 saveTrades UPSERT vs deleteTrade 显式（非聚合根 store 集成路径）

- **聚合根 store**（Watchlist / Workspace）：load() / save(book) · onChange 全量 save 即可 · 删除自动同步
- **非聚合根 store**（Journal trades / journals）：saveTrades([trade]) UPSERT · deleteTrade(id) 显式 · onChange 只能 UPSERT 不能 DELETE · 删除点必须单独调 store.deleteXXX
- **append-only store**（Alert history）：history 由后端 evaluator 自动 append · UI 只读 load · 无 save 路径
- **缓存 store**（KLineCache）：load fast-path + snapshot save / completedBar append · 无 isLoaded 守卫

### 6.6 ChartScene 离线友好设计（M5 第 4 批新引入）

**之前**：snapshot 拉空 → 直接 loadMockFallback（用户看到 Mock）

**现在**：
- snapshot 拉空 + 已有缓存 → 保持缓存 + dataSourceLabel="本地缓存（离线）"
- snapshot 拉空 + 无缓存 → loadMockFallback
- 用户体验：网络中断时仍能看到上次成功拉取的真数据 · 不再 Mock 兜底

## 11 · 可能踩的坑（v11.0 新增 10 个 · v10.0 之前坑保留）

1. **MFI count 单调递增 vs 封顶**（v3 第 12 批）：calculate `for i in n..<count` 跳首窗口 · 增量必须 count 单调递增 + count > period 守卫 · 不能像 BOLL/StdDev 那样 count 封顶 period（第一次实现时全测失败 70 issues）
2. **CMF count 封顶 period vs MFI 单调递增**：calculate `for i in 0..<count` 不跳首 · CMF 用 count == period（同 BOLL/StdDev）· 与 MFI/VR 不同 · 注意区分
3. **VR 三桶 sliding sum + count 单调递增**：同 MFI 模式 · 第 1 根 prevClose=nil → 三桶都 0
4. **VOSC 处理 volume 而非 close**：EMA.makeIncrementalState 用 closes 字段 · VOSC 用 EMA.advance(close: Decimal(volume)) 接口手动 advance · 不能直传 volume KLineSeries
5. **OIDelta state.prevOI 用 Decimal? 而非 Int?**：KLine.openInterest 是 Decimal · KLineSeries.openInterests 是 [Int]（项目历史不一致）· 增量内部统一用 Decimal
6. **PivotPoints defer 写法**：保证 prev 在 return 后才更新 · 避免在 7 列计算前就被覆盖
7. **Supertrend 内嵌 ATR · 共享 history 循环**：makeIncrementalState 内手动构造 KLine 中转推进 atrState + 状态机（同 ChartScene KLine fallback 模式）
8. **Ichimoku CHIKOU 永远 nil**：calculate chikou[i] = closes[i+kN] 用未来 close · 增量协议不支持 · 调用方需用其他途径补
9. **Ichimoku 延迟 ring "读旧写新"**：先读 ring[head]（即将被覆盖 = kN 步前的 raw）· 再写入新 raw · head++ · 不能反顺序
10. **isLoaded 守卫保护 onChange**：SwiftUI .onChange 在 Mock 默认值首次进入 view tree 时会 trigger · 不加守卫会 Mock 写覆盖磁盘 · ChartScene 例外（pipeline 网络数据本身真）
11. **Journal saveJournal 是单个非批量**：协议无 saveJournals · onChange(of: journals) for-loop 逐个 saveJournal · 性能可接受（数据量级小）
12. **Journal deleteJournal 必须显式调 store**：onChange 只能 UPSERT 不能 DELETE · 删除点单独 Task store.deleteJournal(id:)
13. **ChartScene snapshot 拉空 + 有缓存时不退 Mock**：保持缓存 + label="本地缓存（离线）"· 嵌套 if bars.isEmpty 内才退 Mock
14. **maxBars=5000 硬编码**：ChartScene KLineCache append 上限 · 约 21 天分钟线 / 5000 天日线 · M5 上线前评估
15. **simplifier 必须在 build/test 之前跑**（用户多次强调）· 不要 commit 后补 simplifier 双 commit · v3 第 13 批起严格执行新流程
