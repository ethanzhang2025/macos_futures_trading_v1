## 6 · 关键设计沉淀（v9.0 增量 · 跨 commit 共通）

### 6.1 IncrementalIndicator 协议（v9.0 核心）

```swift
public protocol IncrementalIndicator: Indicator {
    associatedtype IncrementalState: Sendable
    static func makeIncrementalState(kline: KLineSeries, params: [Decimal]) throws -> IncrementalState
    static func stepIncremental(state: inout IncrementalState, newBar: KLine) -> [Decimal?]
}
```

设计要点：
- **associatedtype**：每指标自定义 state（MA 用环形 buffer / EMA 用 prev / RSI 用 4 阶段累积 / MACD 嵌套 3 EMA / BOLL 同 MA）· 不在协议层强加统一 state schema
- **不变量**：history.count + step 调用次数 等价于全量 calculate(prefix) · 每步增量结果 == 对应 prefix calculate() 末值（测试断言）
- **流式状态 vs round8 输出**：state 内 prev 不 round（避免误差累积）· 输出 round8（与 calculate 一致 · 测试通过的关键）
- **stepIncremental 不 throws**：参数已在 makeIncrementalState validate · state 内有 invariant 假设
- **静态方法（不是 instance 方法）**：因为 Indicator 协议本身就是 static · IncrementalIndicator 继承

### 6.4 DealCSVParser two-tier error（fileError vs rowErrors）

```swift
enum ImportParseOutcome {
    case success(trades: [Trade], rowErrors: [String])
    case fileError(DealCSVError)
}

static func parseCSV(url: URL, format: DealCSVFormat) -> ImportParseOutcome {
    do {
        let csvString = try String(contentsOf: url, encoding: .utf8)
        let raws = try DealCSVParser.parse(csvString, format: format)   // 文件级 throws
        var trades: [Trade] = []
        var rowErrors: [String] = []
        for raw in raws {
            do { trades.append(try raw.toTrade()) }                     // 行级 throws
            catch let e as DealCSVError { rowErrors.append("第 \(raw.lineNumber) 行：\(e.description)") }
            catch { rowErrors.append("第 \(raw.lineNumber) 行：未知错误") }
        }
        return .success(trades: trades, rowErrors: rowErrors)
    } catch let e as DealCSVError { return .fileError(e) }
    catch { return .fileError(.invalidEncoding) }
}
```

- 文件级失败（编码 / 表头缺列 / 不支持格式）→ 中止 · 整文件 error
- 行级失败（toTrade 字段值非法）→ 累积 rowErrors · 不中止 · 部分成功

## 11 · 可能踩的坑（v9.0 新增 8 个 · v8.0 之前坑保留）

1. **IncrementalIndicator round8 精度对齐**：流式 state 不 round（避免误差累积）· 输出 round8（与 calculate 一致）· **RSI/BOLL 用 round8(prev) 作下游计算输入**（不是用 raw state · 否则末位差 1-2 位）
2. **MACD 内嵌 EMA advance 不能 if-let 链短路**：fast 返回 nil 时 slow 仍必须 advance（state.count 每步递增）· 否则 warm-up 计数错乱
3. **Table 不支持 row-level dropDestination**：WP-43 commit 3 把 Table → List + ForEach + custom Row（commit 1/2 用 Table · commit 3 重构）· 列宽用固定 frame 模拟
4. **ChartIndicatorRunner var struct 在 SwiftUI @State**：mutate 后必须 reassign（"if var runner / runner.step / indicatorRunner = runner"）· 否则 SwiftUI 不刷新
5. **ImportSheet onChange 重解析**：format 切换时必须重跑 parseCSV · 否则用户切换格式但解析结果不变
6. **NSOpenPanel allowedContentTypes**：用 .commaSeparatedText（UTType）· 不是 String "csv"
7. **journals Tab Table contextMenu 多选不显示**：commit 3 设计为单选才显示菜单 · 多选 ≥2 → menu 空（避免批量误删 / 误编辑）· Mac 测试时按 ⌘+Click 多选验证
8. **WatchlistGroupRef vs WatchlistInstrumentRef 同 .data UTType**：SwiftUI .dropDestination(for: T.self) 按 Swift type 区分 · 同 UTType 多类型并存 OK · 但拖错类型不会 false 接受
