## 6 · 关键设计沉淀（v15.0 增量）

### 6.1 simplifier-first 流程强制（v15.0 教训）

**之前模式（v13.31~v13.33 等）**：写代码 → swift build → swift test → simplifier → simplifier 修 → swift build → swift test → commit + push
- 多跑一遍编译/测试 · 浪费

**正确模式（v13.34 起 · 用户严厉强调）**：
```
写代码 → simplifier 复检 → swift build（一次） → swift test（一次） → commit + push
```

- 用户原话："Simplifier 应该在编译之前触发。之前就跟你说了，你没记住吗？"
- 用户后续："你把 Simplifier 的在编译测试前调用写到记忆里 · 强制按这个逻辑执行"
- **已二次强化记忆** `feedback_simplifier流程位置.md`

### 6.2 NotificationCenter 跨模块解耦模式（v13.18）

WP-42 画线 → WP-52 预警联动用 NotificationCenter 而不是直接调用：
- ChartScene 创建 Alert · post `.alertAddedFromChart`
- AlertWindow `.onReceive` 接收 · 防重复 ID 后 alerts.append
- 自动触发 onChange save SQLiteAlertConfigStore + sync evaluator
- 优点：模块解耦 · 多窗口隔离 · 简单防重复

### 6.3 状态持久化矩阵 5 项 + 守卫模式（v15.0）

| 持久化项 | Key | 守卫 | 触发时机 |
|---------|-----|------|---------|
| drawings.sqlite | (instrumentID, period) | isDrawingsLoaded | .task / .onChange |
