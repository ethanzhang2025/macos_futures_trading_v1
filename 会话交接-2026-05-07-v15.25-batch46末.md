## 5 · 关键设计沉淀（v15.24-v15.25 增量）

### 5.1 SyncCore 三层抽象（WP-60）

```
业务模型（Watchlist / WorkspaceTemplate / TradeJournal / Alert / SyncableSettings）
   ↓ 实现 SyncableRecord + SyncRecordDecodable 协议
   ↓ Adapter（toSyncRecord / decode(from:)）
SyncRecord DTO（recordType / id / lastModified / version / deletedAt / payload）
   ↓ SyncEngine actor（pull-merge-push 主循环 + per-recordType baseline）
   ↓ SyncResolver LWW（lastModified 主决胜 + version 副决胜 + tombstone 优先）
SyncBackend 协议（fetch since / push / delete）
   ↓ 实现：MockSyncBackend（内存 · 测试）
   ↓ 实现：CloudKitSyncBackend（macOS-only · #if canImport · Mac 切机配置 entitlements）
   ↓ 实现：AliyunSyncBackend（敏感数据 · Stage B · 留接口）
```

**LWW 简化判定**（防 false positive）：
- 双方 version > 1（实质改过 · 不是初始 v=1）
- payload 不同
- 非双 tombstone
- 才算冲突

**CloudKit 字段约定**（与业务 SyncRecord 对应）：
- recordType / recordName(=id.uuidString) / lastModified(Date) / version(Int64) / deletedAt(Date?) / payload(Data)

**敏感分级（D4 G1 方案 A）**：
- 非敏感（CloudKit）：watchlist / workspace_template / settings
- 敏感（阿里云 Stage B）：journal / alert
- 字段全部预埋 · backend 切换无 schema 改动

### 5.2 iPadApp 平台门控架构（WP-61）

### v15.24 WP-60 新坑

1. **CloudKit container 共享**：iPad 与 Mac MainApp 必须用同 container ID（`iCloud.com.<yourorg>.FuturesTerminal`）· 否则数据不互通

4. **Decimal `1...0` ClosedRange trap**：批 8 / 12 / 29 都遇到过 · 修法 `guard start <= i`
5. **LWW conflict false positive**：单边 baseline 不动 + 双方 v>1 → 误报冲突。生产 wallClock 不会有这问题（batch002 demo 注释已说明）
6. **CloudKit 字段名 < 255 字符 + 不与 CK 系统字段重名**（recordID/recordName/recordType/createdUser/modifiedUser/recordChangeTag）

### v15.25 WP-62 麦语言新坑

1. **无参函数必须带括号**：`TR()` / `OBV()` / `PSAR()` 等 · 不带括号 Interpreter 当变量找
2. **Decimal `1...0` ClosedRange trap**：第 7/12/29 批都遇到过 · 修法 `guard start <= i` 或显式 `if start <= i { for j in start...i }`
3. **Wilder smoothing α=1/N**（不是 EMA 的 2/(N+1)）· 用于 RSI/ADX/SMMA · 与 EMA 是 alpha 不同
4. **SMA(X, N, M) 三参数**：M=1 等价 Wilder · M=2 时 alpha=2/N · 与 trader 标准 SMA(X,N) 不同
5. **Decimal sqrt 走 Double 中转**：精度损失 1e-15 · 与 SQRT/LOG 现有模式一致
6. **CORREL 等浮点比较**：用 ±1e-6 容差 · 不严格 == 1.0
7. **数据不足 N 周期**：MA 在 warm-up nil / EXPSMOOTHING 渐进 · 测试需注意比较起始点
8. **OBV/PVT/NVI/PVI/WAD 累计型**：第一根 = 0 / 1000 等初始值
9. **EXPANDING* 系列**：不限窗口 · 全程累积 · 与 N 周期版本（如 MA）行为不同
10. **GOLDENCROSS/DEADCROSS 边界**：ma1 >= ma2（不严格大于）· prev < (不严格小)
11. **AROONOSC = AROONL - AROONS**（数学恒等式 · 测试已验证）
12. **CYCLE * 100 = STOCH**（数学恒等式 · 测试已验证）
13. **HACLOSE = AVGPRICE = (O+H+L+C)/4**（数学恒等式 · 测试已验证）
14. **VOLATILITYRANK 内部嵌套窗口**：先算 STD（短窗 N/4）再算 PercentRank（N） · 比单层窗口语义更细
15. **SUPERTREND rolling lock**：long 时 ST = max(downBand, prevST) / short 时 min · 否则 ST 错位
