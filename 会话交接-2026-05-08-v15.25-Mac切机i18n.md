## 5 · 关键设计沉淀

### 5.1 i18n 完整工作原理（重要 · 新会话必读）

**SwiftUI 自动 LocalizedStringKey** （零代码改动 · 仅扩 catalog）：
- `Text("中文")` / `Button("中文")` / `Label("中文")` / `.help("中文")` / `.alert("中文")` 等
- SwiftUI 把字面量当 LocalizedStringKey 类型处理 · 默认从 SPM Bundle.module 查 catalog
- 字面量本身就是 lookup key（不需"key.subkey.name"风格）

**String 类型 API** （需用 L() helper · 已批量包装）：
- `panel.title = L("xxx")` / `a.messageText = L("xxx")` / `a.addButton(withTitle: L("xxx"))`
- L() 等价 `NSLocalizedString(key, bundle: .module, comment: "")`
- 定义在 `Sources/MainApp/L10n.swift` · 全局 public func

**.lproj 目录结构**（必须 · 否则 macOS 不识别）：
```
Sources/MainApp/Resources/
├── zh-Hans.lproj/
│   ├── Localizable.strings       (零翻译占位 · key=value 自映射 · 让 macOS 识别 App 支持中文)
│   └── InfoPlist.strings         (CFBundleDisplayName=期货终端 · CFBundleName=期货终端)
├── en.lproj/
│   ├── Localizable.strings       (~765 条 en 翻译)
│   └── InfoPlist.strings         (CFBundleDisplayName=FuturesTerminal · CFBundleName=FuturesTerminal)
└── Localizable.xcstrings         (保留 · 未来 Xcode workflow · runtime 优先用 .lproj)
```

**Package.swift 配置**：
```swift
let package = Package(
    name: "FuturesTerminal",
    defaultLocalization: "zh-Hans",   // 顶层 fallback
    platforms: [.macOS(.v13), .iOS(.v16)],
    ...
    targets: [
        .executableTarget(
            name: "MainApp",
            ...,
            resources: [.process("Resources")]   // SPM 自动识别 .lproj 目录
        ),
    ]
)
```

### 5.2 .app 打包架构（重要 · swift run 不能 i18n）

**问题**：`swift run MainApp` 跑裸 binary · 无 Info.plist · macOS：
- 系统标准菜单（File/Edit/Window/Help）永远 fallback 英文（因为 CFBundleLocalizations 缺失）
- SwiftUI 资源 bundle 路径不对 · catalog 不生效

**解决**：`run-mac.sh` 把裸 binary 包装成完整 .app：
```
.build/{config}/MainApp.app/
├── Contents/
│   ├── MacOS/MainApp                  (binary)
│   ├── Info.plist                      (CFBundleLocalizations=[zh-Hans, en] · CFBundleIdentifier 等)
│   └── Resources/
│       ├── zh-Hans.lproj/Localizable.strings  (从 SPM bundle 拷贝)
│       ├── zh-Hans.lproj/InfoPlist.strings
│       ├── en.lproj/...
│       └── Localizable.xcstrings
```

**用法**：
- `./run-mac.sh` → debug + 启动
- `./run-mac.sh release` → release + 启动
- `./run-mac.sh debug build` → 仅 build · 不启动

### 5.3 Mac 切机修编译关键模式（v15.17 后历史债）

**根因**：`MainApp` SPM target 在 Linux 编译时整 target 跳过（import AppKit 不存在）· 实际未真正 build · v15.17 后 ~485 commit 累积 50 个错误未暴露 · 用户切 Mac 才爆雷。

**4 阶段修复**：
1. **D 类（String→Error）**：`Toast.error("失败", error.localizedDescription)` → `Toast.error("失败", error)`（Toast.error 第 2 参 Error 类型）
2. **H 类（macOS 14+ API）**：`onChange(of: x) { _, new in ... }` 仅 macOS 14+ · 改 `onChange(of: x) { new in ... }` 兼容 13+
3. **C 类（internal 访问）**：`OHLCMarkdownExporter.formatChangePercent` 加 public 修饰符
4. **G 类（NSTextViewDelegate）**：`indexOfSelectedItem index: UnsafeMutablePointer<Int>?` optional 否则协议不匹配
5. **F 类（ViewBuilder 控制流）**：`@ViewBuilder` body 不能含 `var x = 0; for i in ... { x = ... }` · 拆 helper computed property
6. **A 类（state 跨 view）**：`ChartScene.periodShortcuts` 引用 ChartContentView 的 state（measureAnchor/showSwingPoints 等）→ 拆出 `chartContentShortcuts` 移到 ChartContentView
7. **Data Race**：`@MainActor Coordinator` deinit 访问 isolated stored property → 用 `nonisolated(unsafe)` 标记

### 5.4 麦语言第 40-44 批共用模式

每批 7 函数独立文件 `MaiYuYanBatch{40-44}.swift`：
- struct {Name}Function: BuiltinFunction · 实现 execute(args:bars:)
- 内部 helper（如 `maiB40_iq` / `maiB43_ema`）private function · 命名前缀防冲突
- 注册到 `BuiltinFunctions.all` dict（按批次顺序）

**第 40 批 Hilbert** · 简化 IQ：I=X[i]-X[i-3] / Q=X[i-1]-X[i-2]（4 期 90° 相位差近似）
**第 43 批 SCHAFFTC** · 用 Decimal? 替代 `Decimal(Double.greatestFiniteMagnitude)`（防 NaN）
**第 44 批 Butterworth** · Ehlers 公式 c1/c2/c3 系数从 cutoff 周期算

## 10 · 可能踩的坑（本 session 新增）

### 10.1 i18n 坑

1. **swift run MainApp 不能 i18n** · 必须用 `./run-mac.sh` 包成 .app
2. **`@inlinable func` 不能引用 Bundle.module**（internal 修饰）· L() 不能加 @inlinable
3. **SPM 编译 .xcstrings → .lproj 不可靠** · 用 .lproj/.strings 老格式最稳（同时保留 .xcstrings 兼容 Xcode）
4. **catalog 必须给 zh-Hans 显式翻译**（即使等于 key）· 否则 SPM 不生成 zh-Hans.lproj · macOS 判定 App 不支持中文 → 系统菜单 fallback 英文
5. **InfoPlist.strings 必填** · 否则系统菜单的 "About 期货终端 / Quit 期货终端" 无法切英文（因 CFBundleDisplayName 在 Info.plist 写死）
6. **SwiftUI 含插值字符串 normalize 问题** · `Text("\(intvar) 笔")` 内部 lookup key 是 `"%lld 笔"` 不是字面量 · catalog 写错 key 不生效（本 session 跳过 159 条 · P5 处理）
7. **Toast 内部要用 L()** · 否则 `addButton(withTitle: "好")` 永远显示中文（已修）

### 10.2 Mac 切机修编译坑

1. **MainApp Linux 不真编** · target 整体跳过 · 累积 commit 不暴露 → 务必定期切机验证（M5 起每 50 commit 切一次）
2. **ChartScene + ChartContentView state 跨 view** · ChartScene 不能引用 ChartContentView 的 @State · 必须移到 ChartContentView 内部
3. **@MainActor class deinit 访问 isolated property** · 用 `nonisolated(unsafe)` 标记 · 因 NotificationCenter.removeObserver 内部线程安全
4. **Decimal(Double.greatestFiniteMagnitude) 溢出 NaN** · 用 `Decimal? = nil` 替代初始化（batch43 SCHAFFTC 修过）
5. **NSTextViewDelegate `indexOfSelectedItem index` 必须 optional** · `UnsafeMutablePointer<Int>?` · 否则协议不匹配
6. **macOS 14+ API onChange(of:initial:_:)** · 项目支持 macOS 13+ · 必须用 onChange(of:_:) 单参数版

### 10.3 麦语言第 40-44 批坑

1. **stoch / SQUEEZEMOM 用 Decimal? 初始化 hi/lo** · 防溢出 NaN
2. **Hilbert 简化 IQ 仅 4 期相位** · 完整 Ehlers 需 4 阶 IIR + 状态机 · trader 实战足够
3. **OPTIMALF grid search 0.01 步长** · 100 次试算 · 性能可接受
4. **HP_FILTER lambda=0 退化为 X**（α=0）· 测试已验
5. **MARTINGALE 防溢出 streak ≤ 30** · 2^30 ≈ 1e9 已极端
