## 5 · 关键设计沉淀

### 5.1 i18n 三大隐藏陷阱（Batch B/C/D 摸清）

**陷阱 1 · `Text("X \(n)")` 含插值** → SwiftUI 推断 String 不自动 i18n
- 修法：拆 `Text("\(L("X")) \(n)")` 或 `Text(L("X")) + Text(" \(n)")`

**陷阱 2 · `Text(stringField)` 走 String 字段** → 不自动 i18n
- 修法：渲染处包 `Text(L(stringField))` · catalog 加 raw 中文 → 英文翻译
- 例：`Text(template.name)` / `Text(period.displayName)` / `Text(rule.kind.displayName)`

**陷阱 3 · helper 函数 String 参数** → 同陷阱 2
- 修法：调用方包 L()
- 例：`stat("总数", ...)` → `stat(L("总数"), ...)` · `sortableHeaderCell("合约", ...)` → `sortableHeaderCell(L("合约"), ...)`

**enum rawValue 中文** → 增加 `displayName: String { L(rawValue) }` 计算属性 · 用方改 `.rawValue` → `.displayName`
**displayName 计算属性硬编码中文** → switch case 改返回 `L("xxx")`
**跨 SPM module enum**（如 TradingCore.DisciplineRuleKind / Shared.KLinePeriod / Shared.Direction）→ MainApp 渲染处用 `L(rawValue.displayName)` 包一层 · catalog 在 MainApp 加翻译 · 不动 module 内部

### 5.3 公式编辑器栈溢出 hotfix（commit e9cce17）

**根因**：Mac 切机 v15.25 修编译时改 `MaiLangCodeView.Coordinator.textViewDidChangeSelection` 协议 · 引入死循环：
- `makeNSView` line 92 设 `tv.string = text` → AppKit 内部 `setSelectedRange` → 触发 `textViewDidChangeSelection` 通知

**修复**：Coordinator 加 `isApplyingHighlight: Bool` reentrancy guard
- delegate 进入时 `guard !isApplyingHighlight else { return }`
- 调 `applyHighlight` 前后 set/clear flag
- 6 行代码打破死循环

## 10 · 可能踩的坑（本 session 新增）

### 10.1 i18n 系统性陷阱

1. **Text("X \(n)") 含插值** → SwiftUI 推断 String · **必须拆字符串拼接** 才能 i18n
2. **Text(stringField) 走 String 字段** → 不自动 i18n · 渲染处必须包 L()
3. **enum rawValue 走 .rawValue 渲染** → 加 displayName 计算属性 + 用方改 .displayName
4. **跨 SPM module enum displayName** → 不动 module 内部 · 在 MainApp 渲染处包 L() + catalog 加翻译
5. **持久化数据含中文 raw** → 加 L() 包装 · catalog 已知名翻译 · 用户自定义 fallback 原值（`Text(L(template.name))` 模式）
6. **catalog key 与代码字符串必须 100% 一致**（含空格/标点）· 否则 fallback 中文（如 `"持仓时段"` 与 `"持仓时间"` 是不同 key）
7. **改 catalog 后必须 rm -rf .build**（SPM 缓存导致 catalog 不更新）

### 10.2 Mac 切机首通后的 hotfix 模式

1. **Mac 端跑代码前先 `git log -3` 确认拉到最新**（用户曾跑旧 binary 复现旧 bug · UUID 一致铁证）
2. **rm -rf .build 必做**（catalog 资源更新 / 大改动后 SPM 增量编译可能脏）
3. **crash log 路径**：`~/Library/Logs/DiagnosticReports/MainApp-*.ips`（macOS 13+ 新格式 · JSON 双段）
4. **栈递归类崩溃** → 看 `recursionInfoArray` 关键 frame 的重复模式 · 找 reentrancy guard 缺失点

### 10.3 自动化测试模式

1. **screencapture -W 后 ESC 取消** → 脚本会自动 retry 一次 · 第二次还失败跳过
2. **截到 Terminal 自身** → 用户必须先在 App 内打开/激活窗口 · 再回 Terminal 按回车
3. **部分模式不清远端旧图** → 我看的是单/多/范围更新后的最新组合
4. **9 窗口对照表必给**（用户问"06 是哪个" → 不能让用户查脚本）

### 10.4 IAP / Pro 门控陷阱（M6 前不做）

1. **不要现在写 SubscriptionGate 抽象层** · YAGNI · 等真做时再写
2. **不要硬编码 Pro 标记到具体 view 内部** · 保持模块化 · 入口加 gate check 即可
