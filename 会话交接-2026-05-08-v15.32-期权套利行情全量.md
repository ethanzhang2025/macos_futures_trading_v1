   ↓                     ↓                     ↓
OptionStrategy ── OptionStrategyBuilder ──> 5 经典策略

## 10 · 可能踩的坑（本 session 新增）

### 10.1 期权 Phase 1 数据层

1. **OptionContract 暂不 Codable**：因 Exchange enum 不 conform · v1 跳过 · v2 接 CTP 持久化时扩
2. **ETF 期权延后**：50ETF/300ETF 需要 SSE/SZSE 交易所 enum · v1 仅 IO/m/SR 3 旗舰
3. **美式期权用 BS 近似**：豆粕/白糖到期溢价高估 · v3 接二叉树
4. **strike step 各品种不同**：IO/m=50 · SR=100 · 改 strike 别忘看 step

### 10.2 期权 Phase 2 Greeks 数学

1. **Theta 单位**：BS 公式输出**每年** · UI 显示需 ÷ 365 转每日
2. **Vega 单位**：BS 输出每 100% 波动 · UI 报告每 1% 需 ÷ 100
3. **σ ≤ 0 / T ≤ 0 / S ≤ 0** → 退化为内在价值（不严谨但稳健 · 主流券商也这么处理）
4. **PCP Theta 推导**：`Theta_call - Theta_put = q·S·e^(-qT) - r·K·e^(-rT)`（不是反过来！我自己第一次写测试就反了）

### 10.3 期权 Phase 3 IV 反推

1. **Newton 收敛快但不稳**：vega 接近 0（深 OTM/末日）失败 · 自动让 bisection 接管
2. **市价 < 内在价值** → 套利违例 · 返 nil（不是修正后试）
3. **bisection 区间** [0.001, 5.0]：覆盖 0.1% ~ 500% 隐波 · 超出 → nil
4. **T=0 + 市价 ≠ 内在值** → nil（市价合法性 = 严格内在值）

### 10.4 期权 Phase 4 策略

1. **Long Butterfly 3 strike 等距**：mid - low = high - mid · 否则 PnL 不对称
2. **数量 quantity 始终正**：方向由 direction 决定（long/short）
3. **breakeven 二分**：相邻 strike 间符号变化 → 二分 · 多个零点全列出
4. **远端无限性判定**：在 spot=1e6 / 0.01 处评估 · 与中段 max/min 比较

### 10.5 期权 Phase 5 UI

1. **OptionWindow 默认大小** 1180×760 · 小屏幕可能挤
2. **现价/利率/IV 三个 TextField**：用户修改后 chain 不重算（chain 是 sampleChain · 不依赖现价）· Greeks 实时刷
3. **5 策略需要的 strike 数不同**：长跨式只用 mid · 牛熊用 low+high · 蝶式用 low+mid+high
4. **strike 字段都是 3 个**（low/mid/high）· 不需要的字段被忽略 · UI 统一简化
