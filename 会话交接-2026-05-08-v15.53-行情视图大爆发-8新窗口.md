### 5.3 关联性矩阵 + 板块因子 mock（v15.48）

**Sources/Shared/Correlation/**：
- `CorrelationCalculator.swift` · 皮尔逊 r 公式 + log-return 转换
- `CorrelationMockSeries.swift` · 板块因子注入（同板块 r 0.3-0.7 · 跨板块 < 0.4）

**关键算法**：
```swift
价格 = exp(板块因子 × 0.65 + 独立扰动 × 0.35) · 累积 random walk
板块因子：每板块独立 seeded 时序（同板块共享）
独立扰动：每品种独立 seed
```

**自然产生差异化相关性**：trader 看到"黑色系内部强相关 · 跨板块弱相关"等典型形态。

### 5.4 跨期套利数据层（v15.50）

## 10 · 可能踩的坑（本 session 新增）

### 10.1 视觉收敛
- **macOS-only 字段误用易遗漏**：v15.40 hover 我用错了 `optionPnL`/`underlyingPnL`（实际是 `optionMTM`/`underlyingMTM`）· `OptionStrategyAnalysis`（实际是 `PayoffAnalysis`）· `leg.strikePrice`（实际是 `leg.contract.strikePrice`）· Linux 不编译该段所以 build 不报错 · Mac 编译才暴露
- **修复方式**：现场修正后 commit · 但写新 SwiftUI 代码时务必跑下 swift build 看 Mac 端是否真编译

### 10.2 IV smile（v15.42）
- mock 公式：iv(K, S) = baseIV × (1 + skew × ln(K/S)² × 4) + put_skew_offset
- 经典股票笑容：左翼（K<S · put 端）IV 高 · 右翼（K>S · call 端）IV 低
- v2 接 BlackScholes.impliedVol 反推（接口预留）

### 10.3 关联性矩阵（v15.48）
- mock 板块因子注入：sectorRatio = 0.65 → 同板块 r ≈ 0.6-0.8 · 跨板块 r ≈ 0-0.3
- N×N grid 性能：60×60 = 3600 cell · 每次重绘重算 ≈ 几十 ms · 没缓存（trader 体感未必卡 · 但若卡 Mac 验收时加 cache）
- cell ≥ 32pt 才显示 r 值 · 否则字会糊

### 10.4 跨期套利（v15.50）
- mock 公式：远月 = 近月 + 持有成本（≈ 1.5%）· 大部分 contango 主导
- contango/backwardation 自动判定阈值：spread > 1 → contango · spread < -1 → backwardation
- seeded RNG 含 Date()：时间戳不可复现（已加测试豁免 · 仅比价格序列）

### 10.5 品种深度分析（v15.51）
- **必须缓存**：cachedSeries / cachedRelated · 否则 hover 时反复跑 60×60 corr 几十 ms · Mac 上肯定卡
- @State + onChange(selectedID) + onAppear 触发 refreshCache · 已加

### 10.6 跨窗口联动（v15.53）
- HeatmapWindow 不切 sortMode · 仅加 highlightedID（避免破坏用户当前看的视图）
- CalendarSpreadWindow 没加联动 · 因 ID → 跨期对的映射间接（RB0 ↔ RB05/RB10 跨期）
- 联动后旧 sectorFilter 被覆盖 · 用户可能不期望

### 10.7 测试节奏强化规则
- v15.45 起：commit 前 5 步决策树 · 不再默认全量
- 触发全量的真实场景：新增公共 API（v15.43/48/50）/ 跨 ≥3 module
- 跳全量场景：纯 macOS-only SwiftUI（约一半 commit 都是这种）
