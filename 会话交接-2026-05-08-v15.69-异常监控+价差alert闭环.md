## 10 · 可能踩的坑（本 session 新增）

### 10.1 spreadDeviation alert 数据契约（v15.57）

- **AlertCondition 加新 case 触发多处 exhaustive switch**：v15.57 batch1 改了 5 处（onTick/onBar/HistoryFilter/2 个 CSVExporter）+ AlertWindow ConditionKind/loadConditionParams/toCondition/testTriggerPrice 共 9 处
- **sheet picker 隐藏 spreadDeviation**：用 sheetEditableCases · 仅 ⌘⌥W + 按钮入口创建（防误用）
- **Codable 兼容**：旧 JSON 不含 spread case · 新版兼容反序列化（Set 安全）

### 10.2 AlertEvaluator.onSpreadValue 入参 [Decimal] 设计（v15.60）

- 最初设计 SpreadValueLike protocol · v15.60 batch2 简化为 [Decimal]（KISS）
- caller `series.map(\.value)` 一行转换 · 比 protocol 抽象更直接
- AlertCore 不引 DataCore 反向 dep

### 10.3 SpreadAlertWindow timer 60s 周期（v15.60）

- `.task` 启动 `runEvaluatorPushLoop` · `while !Task.isCancelled` + sleep
- 26 对每次扫描 ~50ms（mock seeded RNG · 200 点）
- v2 真行情接入时改：从 SpreadValue stream 直接 push（无需 timer）

### 10.4 spread alerts 持久化竞态（v15.67）

- SpreadAlertWindow 写 alertConfig store + post NotificationCenter
- ⌘B 已打开时 onChange(alerts) 也 save · 两次 save 内容一致（actor 串行）
- ⌘B 未打开时 SpreadAlertWindow 直写 alertConfig · 下次 ⌘B 打开 .task load 含此条
- 防重：persistSpreadAlert 检查同 spreadID + isCalendar 已存在则不 append

### 10.5 HUDFieldsBook default 改动（v15.58）

- 既存用户 UserDefaults 已写盘的偏好不受影响
- 仅 v1 新装 / "还原默认" 按钮 / 首次启动时生效
- 测试 `defaultExcludesNewFields` 仍通过（v15.20 字段确实不在 default）

### 10.6 sparkline Canvas 渲染（v15.59/68）

- 30 cell mini bar chart · 全局 peak 归一化（行间高度可比）
- v15.68 spread sparkline · 上下轨 dash + mean 灰虚线 + 末点 z 色阶高亮
- Canvas 闭包不能持有 self 强引用（捕获不当会内存泄漏）· 实际用 let 局部 OK

### 10.7 测试节奏 · CalendarSpread flaky（v15.60）

- swift testing 全量偶尔 1 issue（"CalendarSpreadCalculator · mock 时序"）
- 原因：String.hashValue 跨进程随机 · seeded RNG 衍生数据浮动
- 解：重跑（实测一次重跑就过）· 长期：用 FNV-1a 等稳定 hash · 暂搁

### 10.8 AlertCondition 跨模块 switch（v15.63）

- AlertCondition 是 AlertCore public enum · MainApp extension 内 switch 不强制 exhaustive（non-frozen 跨模块特性）
- 缺 case fall through 不报错 · 但运行时返回 fallback 值
- v15.63 显式加 .spreadDeviation case 处理 · 避免 spread alert 显示 ""
