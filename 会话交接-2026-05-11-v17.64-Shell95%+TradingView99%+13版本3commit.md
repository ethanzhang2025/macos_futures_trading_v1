## 5 · 工程踩坑（本 session 实测）

### 5.1 ShellViewModel 提到 App 级 · @StateObject vs @EnvironmentObject
- ShellWindow 原 `@StateObject private var shellVM = ShellViewModel()`
- 改 FuturesTerminalApp `@StateObject` · ShellWindow / DetachedPaneWindow 用 `@EnvironmentObject`
- 关键：两个 WindowGroup 都注入 `.environmentObject(shellVM)` · 跨窗口共享 ✅

### 5.2 SwiftUI WindowGroup(for: String.self) 多窗口
- `openWindow(id: "detachedPane", value: paneID.uuidString)` 触发
- 用 String（UUID.uuidString）传 Pane ID · 比 PaneConfig 直接传值简洁
- 窗口内反查 shellVM.workspaces 找 PaneConfig · 找不到时显示"已不存在 · 关闭"提示

### 5.3 ChartScene 跨 module 通信用 Environment closure
- ChartScene 不能直接持 shellVM（ChartScene 也用于独立窗口）· 用 `@Environment(\.shellCrosshairReporter)` 注入 closure
- PaneHost 注入：`{ paneID, date in shellVM.setPaneCrosshair(paneID:, unixTime:) }`
- ChartScene hover handler 调 reporter(id, bars[barIndex].openTime) · 解耦

### 5.4 PaneConfig UUID round-trip 防冲突
- 导入 Workspace JSON 时：ws.id + 所有 panes.id 重生 UUID · 防与现有 Pane 主键冲突
- group color 也清空 · 防与现有 group 互相污染

### 5.5 SidebarLayoutStore 缺失/多余 case 兼容
- 未来新增 SidebarSection enum case · 老 JSON 缺失字段 → 自动追加到末尾仍可见
- 老 JSON 含已删除 case · 过滤掉避免渲染崩

### 5.6 NSSavePanel / NSOpenPanel UTType
- 需要 `import UniformTypeIdentifiers` · `allowedContentTypes = [.json]`
- WorkspaceTabBar 已加 import

### 5.7 ChartScene @Environment 顺序（避免 displayBars 类型推断爆炸）
- 加 4 个新 Environment（shellPaneID / reporter / externalCrosshair）放在 isHostedInShell 后面
- displayBars switch 多 case 改时 Mac 编译可能爆类型推断 · 用 `()` IIFE 包

### 5.8 SubChartView ComputeKey 加 vpModeRaw
- 切 VP mode 时 task(id:) 需要 trigger 重算 · ComputeKey 必须含 mode raw value

### 5.9 ReplayDriverTests "跑到末尾自动停" 偶发 flaky
- Timer 驱动 · 系统 schedule 抖动 · 单跑 0.301s pass · 全量并跑可能 0.7s+ 失败
- 与本 session 改动无关 · 不动
