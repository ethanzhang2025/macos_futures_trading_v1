## 10 · 本 session 学到的（写入下次会话提醒）

### Memory 决策反转：feedback_战略不补 中"艾略特国内小众"作废

v17.128 用户主动要求补艾略特 · `D3-风险与危机预案.md` §4 战略不补表中"艾略特工具 · 国内小众"明确推翻。

**经验**：用户主动要求时 · 不要拿"memory 说不做"挡回去 · 反问"是确定这次要做吗" · 用户答确认就开做（同时清理 memory 决策表）。

### DrawingType 加 case 涉及多 module 触点

每加一种 DrawingType case 需要至少 6 处更新：
- Shared/Drawing.swift（case + pointsNeeded + factory）
- ChartScene.swift（displayName + category + toolbar + preview + hit-test 4-5 处）
- ChartCore/DrawingsOverlayView.swift（render dispatch + 实装方法 + 默认色）
- Tests/SharedTests/DrawingsTests/DrawingsTests.swift（pointsNeededContract + roundTrip 共 2 处）

**最容易漏**：Tests 的"22 类"硬编码 · 加 case 后必须同步 22 → N。本 session 漏了 1 次（v17.126-128 加 5 case · 测试期望未改 · v17.131 修）。下次加新 DrawingType 必须**同 commit** 改 DrawingsTests 期望值。

### MainAppTests target 跨 executableTarget 测试 pattern

新建 SPM testTarget 测试 `.executableTarget` MainApp 是 **罕见但可行** pattern：
- `.testTarget(name: "MainAppTests", dependencies: ["MainApp", "Shared"], path: "Tests/MainAppTests")`
- 测试源文件用 `@testable import MainApp` 访问 internal 符号
- Linux 端 `#if canImport(SwiftUI) && os(macOS)` 守卫整 file · target 空跑（0 tests）
- macOS 端 SPM 自动解决 @main 入口冲突 · executableTarget 单独编译 + testTarget link 时跳过 @main

**注意**：linux 上 swift test 显示"No matching test cases" 但 build 成功 · 不是错误 · 是 #if guard 设计如此。

### Watchlist Codable 加字段必带 decodeIfPresent 守

数据模型扩展（v17.131 加 sortFieldRaw/sortAscending · v17.36 加 colorIndex）必须：
- struct 字段用 `var x: T?`（可为 nil）
- init 默认 nil
- CodingKeys 加 case
- decode 用 `decodeIfPresent` 守
- encode 用 `encodeIfPresent`
- 单测覆盖旧 JSON decode 兼容

**测试模板**（v17.131 引入）：用字符串构造模拟旧 JSON 直接 decode 验默认值。

### #expect 不能直接调 mutating 方法

`#expect(book.setGroupSort(...))` 编译错 · macro 把 book 当 immutable 捕获。
**解决**：先 `let ok = book.setGroupSort(...)` 然后 `#expect(ok)`。

### batch 聚类节奏：测试期望值修复同 commit

按 memory `batch聚类节奏` · v17.131 把"修画线测试期望 22→27"与"自选每组排序"塞同一 commit · 测试一次跑全绿。
**经验**：连续 batch 间发现的 fallout test failure · 直接收口在当前 batch · 不另开 commit。
