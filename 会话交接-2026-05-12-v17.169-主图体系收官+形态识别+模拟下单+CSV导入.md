## 4 · 本 session 关键产出

### A · 主图 overlay 增量化体系（v17.155-157）

**Why**：v17.139 step(newBar) 全量 MainChartOverlayCompute.compute(historyBars+newBar) · 5kbars 9 overlay ≈ 50ms/根 · 8× 回放速度卡顿。

**v17.155 SAR + SuperTrend IncrementalState 补齐**：
- IndicatorCore 之前已有 8 个 overlay 的 IncrementalIndicator · 唯独 SAR + SuperTrend (v17.139 新版) 缺失
- SAR：状态机 isLong/ep/af/sar + prev/prevPrev high/low buffer · count==1 缓冲首根 · count==2 seed isLong + 跑 i=1 迭代 · count>=3 跑 i>=2 双层 prev/prevPrev 限制
- SuperTrend：内嵌 ATR.IncrementalState + prevST(round8) + prevClose + trendIsLong + firstValidSet · 输出 [SUPERTREND, SUPERTREND-DIR] 2 列
- 8 单测（4 SAR + 4 SuperTrend · history 满 / 空 / 1 根 / 参数错抛错）

**v17.156 OverlayIncrementalStates 聚合**：
- 拆 MainChartOverlayCompute 出 ChartScene.swift（fileprivate macOS-only）· 现独立 Sources/MainApp/OverlayIncrementalStates.swift · 跨平台 Linux 可测
- OverlayIncrementalStates struct · 9 overlay 全 IncrementalState · init(history:book:) + step(newBar:) -> [Decimal?]
- 列顺序与 MainChartOverlayCompute 完全对齐（关键：KC step 返回 [mid,upper,lower] · compute 重排 [upper,mid,lower] · 此处映射 row[1]/row[0]/row[2]）
- ChartIndicatorRunner.step 改 overlayStates.step 增量推进 · 删 historyBars + overlayBook 字段
- 12 单测：9 单 overlay parity + 12 全开列顺序 + 混合启用 + 空 book

**v17.157 perf baseline**：
- prime 5000 history + step 500 with 12 overlay 全开 · Linux Decimal soft 中位 ~4s（Mac 预期 5-10× 更快）
- 单 step 中位 ~1.5ms · 已远低于 v17.139 全量 50ms+ 等级
- 阈值 60000ms / 100ms · 2× safety · CI 不同机器都能过 · 防未来误改回全量

### D · 🥇 形态识别（v17.163-165）

**v17.163 PatternDetector 算法核心**：
- IndicatorCore/PatternDetector.swift · 4 形态：头肩顶/底 + 双顶/底
- 算法栈：ZigZag(3%) 摆动 → pivot 序列 → 滑动 3/5 窗口模板匹配
- PatternKind enum · displayName/direction/icon
- DetectedPattern struct · kind/pivotIndices/pivotPrices/confidence(0..1)
- PatternDetectorParams：zigzag 阈值 + 4 容忍度（肩对称 / 头突出 / 颈线 / 双顶 + 中间回撤）
- 4 形态匹配模板：
  · 头肩顶：5 pivot 高低高低高 · 头 > max(肩) × (1+3%) · 肩对称 ≤10% · 颈线对齐 ≤10%
  · 头肩底：mirror
  · 双顶：3 pivot 高低高 · 顶差 ≤3% · 中间回撤 ≥2%
  · 双底：mirror
- 重叠去重：同 endIndex 保留 confidence 最高
- confidence = 加权得分（对称 0.4 + 颈线 0.3 + 头突出 0.3 / 双顶: 对齐 0.6 + 回撤 0.4）
- 12 单测覆盖（4 形态精确 + 阈值边界拒识 + 空 / 单调 / 去重 / 排序）

## 10 · 本 session 学到的（写入下次会话提醒）

### IncrementalState 移出 fileprivate 才好测（v17.156）

OverlayIncrementalStates 原本写 ChartScene.swift fileprivate · 测试不可见。
拆到独立文件 Sources/MainApp/OverlayIncrementalStates.swift（无 SwiftUI 依赖）· 跨平台 Linux 可测 · MainAppTests 加 @testable import MainApp。
未来类似聚合状态都按此模式：「无 UI 依赖的状态聚合 = 独立文件 + internal access」。

### 列顺序对齐 = 全量 calculate vs 增量 step 必须一致（v17.156）

KC IncrementalIndicator step 返回 [mid, upper, lower] 与 calculate 一致 · 但 MainChartOverlayCompute 重排为 [upper, mid, lower]。
OverlayIncrementalStates.step 必须做同样重排（row[1]/row[0]/row[2]）· 否则 step 与 compute 末值不一致。
模板：每加新 overlay 必看 IndicatorCore.calculate 返回顺序 vs MainChartOverlayCompute 取列顺序。

### 退避填充模式（v17.161 CHIKOU）

某些指标的"当前根"值依赖未来 K · 增量永远 nil。但可以**退避填充上游位置**：当 newBar 进来 · 把 close 写到 (newLen-1-kijun) 历史位置。
等价 calculate 的 shiftBackward。模板：未来依赖的指标都可这么做（如果有 kN 已知的 shift 量）。

### MainChartOverlayBook 加新 Bool toggle 字段模板（v17.161/162）

每加 toggle：
1. struct 加字段 + default · init 加参数
2. CodingKeys 加 case · decodeIfPresent fallback
3. 加 1 fallback test：用旧版 init 构造 encode · 新版 decode 验 fallback default
4. 加 1 round-trip test

### TradingCore SimulatedTradingEngine 是宝藏（v17.167）

506 行完整 actor · 含 submitOrder/cancelOrder/onTick/snapshot/observe/DisciplineRule。
未来"模拟下单相关"功能都先 grep TradingCore 看是否已有 · 别重复造轮子。
ChartScene 调用模式：`@Environment(\.simulatedTradingEngine)` + `Task { let (ref, rej) = await engine.submitOrder(...) }`。

### SwiftUI sheet + Linux build（v17.165/169）

新增 sheet 文件必须 `#if canImport(SwiftUI) && os(macOS)` 守 · 否则 Linux build fail。
KLineCSVImportSheet / PatternsListSheet 都按此模板。
Test 文件如果 @testable import MainApp 的 macOS-only 类型 · 也要同样守。

### CSV 时间格式嗅探（v17.169）

7 种 candidate 顺序：精确长格式优先 · yyyy-MM-dd HH:mm:ss > yyyy-MM-dd > yyyyMMdd HHmmss > yyyyMMdd > Unix。
前 5 行任意成功即锁定 · 避免歧义（比如 "20260512" 既是 yyyyMMdd 又可能是 unixsec 但范围远小于真实秒数）。

### 4 形态算法心得（v17.163）

- ZigZag percent=3% 适中 · 太小噪声多（每根都是 pivot）· 太大错过形态
- 头肩对称 10% / 头突出 3% 容忍 trader 实际场景适用
- 双顶差距 3% / 中间回撤 2% · 严一点 trader 不会被假信号污染
- confidence 加权 = 对称 + 颈线 + 头突出 各 0.3/0.4 · 比硬阈值好
- dedupByEndIndex：同 endIndex 多形态命中（如 5 pivot 同时是 HS + 内含双顶）· 保留 confidence 最高

### 形态/支撑阻力 overlay 渲染统一模板（v17.164/166）

- 基于 visible 区间过滤 · 避免大量历史 marker 污染视图
- GeometryReader + 公共 hi/lo/span 算 y 坐标
- 多绿空红用 candleUp/Down(mode: candleColorMode)（与全局配色一致）
- 不拦截 hover（allowsHitTesting false）· 与 swing/drawings 同模式
- ⌘⇧X 快捷键 + GlobalShortcutsCatalog 注册
