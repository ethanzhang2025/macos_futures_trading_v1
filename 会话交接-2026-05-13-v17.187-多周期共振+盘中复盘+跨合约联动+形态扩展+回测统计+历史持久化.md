### dedupByEndIndex 改 pivotCount 优先（v17.173 关键改进）

旧规则：同 endIndex 仅留 confidence 最高
新规则：先按 pivotIndices.count 优先（结构更长更具体）· 同长度再按 confidence
原因：4-pivot 上升三角的内含 3-pivot doubleTop 同 endIndex · 偏好三角（更完整结构）
教训：测试数据要小心同 endIndex 多形态命中 · doubleTop confidence 通常较高会"吃掉"4-pivot 三角

### EMA cross 测试要用非线性数据（v17.170 教训）

线性匀速升降时 · DIF 在 EMA(2)-EMA(4) 等组合下会严格 == DEA（在 round8 精度内）
strict cross 不触发 · 要用加速序列（如 quadratic）让 DIF ≠ DEA

### CrossInstrumentLinkage UI 接 DataCore（v17.178 教训）

MainApp 默认含 DataCore 依赖 · 所以 `import DataCore` 直接可用
但 Shared 模块不能 import DataCore（向下依赖）· 跨合约核心要保留在 Shared

### UserDefaults JSON 上限考量（v17.187）

CrossLinkageObservationHistory 默认 1000 条上限 · 经验：1000 个简单 struct < 100KB
JSON 写入 UserDefaults 失败静默 · 与 IndicatorFavoritesStore 同模板
不要无脑超过 10000 条 · 接近 1MB 会有性能问题

### simplifier 5 版本规则（v17.174 + v17.179）

L1 必跑（actor/Codable/CSV/算法）· 本 session 全 L1 · 跑了 2 次
关键发现：注释"为什么"留 · WHAT 删 · guard 链合并不动语义
不要无脑合并 confidence 公式（权重不同的形态合 helper = 过度抽象）

### SwiftUI sheet + Linux 守（v17.171/184/185 都遵循）

新 sheet 文件必须 `#if canImport(SwiftUI) && os(macOS)` 守
否则 Linux build fail · MainAppTests 引用时同样守

### OHLCMarkdownExporter 不依赖 IndicatorCore（v17.185 设计原则）

Shared 模块保持独立 · 不上调依赖 IndicatorCore
caller 自己跑 PatternDetector · 拿到 patterns 后构造 annotations dict 传入

### ChartScene 已有 5900 行（v17.184 后）

继续加新功能时优先：
1. 抽到独立 Sheet/View 文件
2. 抽到 helper 函数（保留为 private member）
3. 实在简单可以直接加 @State + 小段 view
