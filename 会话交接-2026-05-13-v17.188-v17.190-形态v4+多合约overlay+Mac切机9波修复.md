## 6 · 重点提醒（写入新会话）

### 6.1 Mac 切机首次暴露的设计问题

ChartScene.swift 文件**结构性问题**：
- ChartScene struct (line 153-2607) 与 ChartContentView struct (line 2611-5995) 共存于同一文件
- ChartContentView 内有大量 view computed property（chartContentShortcuts / viewportShortcuts / secondaryInstrumentOverlay）引用 ChartScene 的 @State / method
- 历史 refactor 把 ChartScene 的某些 view 移到 ChartContentView 但 @State / @Environment / private func **没同步移过去**
- Linux 不编译 SwiftUI 守内代码 · 漏检 ~400 commit
- 第 12 波修复**只是补 ChartContentView 缺失 state · 不是根本重构**

**新会话警示**：每次给 ChartScene.swift 加新 @State / @Environment / private func 时 · 必须**双向检查**它会被 ChartScene 用还是 ChartContentView 用 · 严格遵守"在哪个 struct 用就定义在哪个 struct"。

### 6.2 Mac swift 6.3.2 严格规则（已记 memory）

按 memory `feedback_Mac_swift63严格三类陷阱` + 本 session 扩展：

| 规则 | 触发 |
|---|---|
| covariant Self 不能在 stored property initializer | `Self.redColor` → `MetalKLineRenderer.redColor` |
| StrokeStyle 参数序 lineCap 必先于 dash | macOS 26 严格 |
| onChange(of:initial:_:) macOS 14+ | 用单参数版本 |
| Identifiable required for SwiftUI Table | 加 `Identifiable` + `id` 计算属性 |
| ViewBuilder buildBlock 最多 10 children | 超过拆 inner Group |
| @ViewBuilder 内禁 statement（var.append / nested func / explicit return） | 提到 helper function |
| sheet callback closure 字段须 @MainActor | onConfirm/onClear/onJumpTo/onApply 字段标注 |
| actor 内 public static let @AppStorage 不识别 | 外置 enum / 用 nonisolated |
| `cannot find in scope` 级联 | 通常是同 struct 内某 syntax/buildBlock error 让 swift 退出 struct body parsing · 后续所有 stored property/method 都 cannot find |

## 10 · 本 session 学到的（写入新会话）

### Linux SwiftUI 编译漏检（最大教训）
- `#if canImport(SwiftUI) && os(macOS)` 守内代码 Linux 完全不编译
- 整个 ChartScene.swift（~6020 行）+ 所有 macOS-only window 文件 = Linux 看不见的"暗物质"
- 累积 ~400 commit 没切机 = 暗物质里累积 ~30+ Mac 严格错
- **教训**：v15.82 以来切机间隔太长 · 必须缩短切机周期（v17.190 之后建议 5-10 commit 切一次）

### Mac extract 进化
- v1（极简）：仅 grep `error:` 行 · 看不到 swift compiler 的 note 真根解释
- v2（带 context）：每条 error 后 3 行 · token ~3KB · 第 11 波后定位 ChartContentView 真根
- **新会话默认用 v2**

### ChartContentView vs ChartScene 结构性问题
- 同一文件两个 struct · 内部 view 互相引用对方 state
- 历史 refactor 没保持 single-source-of-truth
- 第 12 波只是补 state 让 Mac 编译过 · 真重构（合并 struct 或明确边界）是 v18+ 任务

### dedupByEndIndex 改 pivotCount 优先（v17.173 关键 · v17.188 沿用）
- 4-pivot 三角内含 3-pivot 双顶 · 偏好三角
- v17.188 加 4 形态全 5-pivot · 不冲突

### MultiInstrumentNormalizer firstBaseline 等价百分比
- secondary 涨 N% → 映射到 primary[0] × (1+N%)
- 验证：[50, 55, 60] 映射到 primary[0]=100 = [100, 110, 120]（精确等价）

### ViewBuilder buildBlock10 上限
- SwiftUI Group / @ViewBuilder method 最多 10 child views
- 超过 swift 6.3 严格 generic 推断 fail → 整个 struct cannot find 级联
- 拆 inner Group 分组
