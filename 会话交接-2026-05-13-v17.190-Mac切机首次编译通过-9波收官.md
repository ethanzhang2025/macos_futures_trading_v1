## 5 · 学到的（写入新会话提醒）

### 5.1 Linux SwiftUI 编译漏检（最大教训 · 整个 session）

- `#if canImport(SwiftUI) && os(macOS)` 守内代码 Linux 完全不编译
- ChartScene.swift（~6020 行）+ 所有 macOS-only window 文件 = Linux 看不见的"暗物质"
- v15.82 → v17.169 累积 ~400 commit 没切机 = 暗物质内累积 ~30+ Mac 严格错
- **教训**：切机间隔太长是巨大风险 · 建议 **5-10 commit 切一次**（v17.190 之后必须做到）

### 5.2 Mac swift 6.3.2 严格规则（已扩展 memory `feedback_Mac_swift63严格三类陷阱` 到 9 类）

| 规则 | 触发 / 修法 |
|---|---|
| covariant Self 不能在 stored property initializer | `Self.redColor` → `MetalKLineRenderer.redColor` |
| StrokeStyle 参数序 lineCap 必先于 dash | macOS 26 严格 |
| `onChange(of:initial:_:)` macOS 14+ | 用单参数版本（macOS 13 target） |
| SwiftUI Table 需要 Identifiable | 加 `Identifiable` + `id` 计算属性 |
| ViewBuilder buildBlock 最多 10 children | 超过拆 inner Group |
| `@ViewBuilder` 内禁 statement（var.append / nested func / explicit return） | 提到 helper function |
| sheet callback closure 字段须 @MainActor | onConfirm/onClear/onJumpTo/onApply 字段标注 |
| actor 内 public static let `@AppStorage` 不识别 | 外置 enum / nonisolated |
| `cannot find in scope` 级联 | 通常某 syntax/buildBlock 错让 swift 退出 struct body parsing · 后续所有 stored property/method cannot find |
| ChartContentView vs ChartScene 同文件双 struct 互引用 state 不一致 | 各 struct 自包含 |

### 5.3 mac_acceptance.sh extract 改进 v2

`01_build_error_extract.txt` 现含每条 error 后 3 行 context（含代码 + caret 标记）· token ~3KB · 让远程 Claude 一次定位真根。

**这是第 11 波关键升级**：v1 仅 grep error: 行（看不到 swift compiler 的 note: 真根解释）· v2 含 context 让我在第 12 波一次找到 ChartContentView 真根。

### 5.4 ChartScene.swift 结构性遗留问题（v18+ 重构候选）

- ChartScene struct (line 153-2607) 与 ChartContentView struct (line 2611-5995) 共存同文件
- ChartContentView 内有大量 view computed property（chartContentShortcuts / viewportShortcuts / secondaryInstrumentOverlay）
- 第 12 波只补 ChartContentView 缺失 state · **不是真重构**
- v18+ 建议：明确 struct 边界 · 抽 SubChartView / ChartShortcutsView 等独立 file
