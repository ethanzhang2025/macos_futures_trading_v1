## 3 · 4 大真修详解（最重要）

### 3.1 v17.196 真修 · ChartContentView 9 @State → @Binding

**真根因**：v17.190 第 12 波"假修复"= 为让 Mac 编译过，在 ChartContentView 补 9 个同名 @State。但 ChartScene 已有同名 @State + sheet/overlay modifier。两个 struct 各自独立 state · 完全不连通。

**功能损坏（用户实测）**：
- ⌘⇧L 形态清单 sheet 永远打不开（state 写 ChartContentView · sheet listen ChartScene）
- ⌘⌥G 多合约 picker sheet 永远打不开
- ⌘⌥⇧Y 共振统计 sheet 永远打不开
- v17.189 多合约 overlay 永远不渲染（picker onApply 写 ChartScene secondaryBars · ChartContentView body 读自己空 secondaryBars）

**修法**（commit 5bb0693 + 6586679 hotfix）：
- ChartContentView line 2725-2734 9 @State 全改 @Binding
- ChartContentView 显式 init 补 9 个 Binding 参数（line 2806）
- ChartScene mainContent 实例化 ChartContentView 时传 9 个 `$xxx`
- single source of truth = ChartScene

### 3.2 v17.197 真修 · 3 个快捷键同 file 冲突

**真根因**：ChartScene.swift 内 3 对 keyboardShortcut 重复绑定 · SwiftUI 优先匹配靠前的 hidden Button · 低频功能 guard 失败时静默 → 高频完全无反应。

| 冲突键 | 高频（保留原键） | 低频（加 option 改 ⌘⌥⇧X） |
|---|---|---|
| ⌘⇧L | v17.165 形态清单 sheet | v13.28 画线锁定（guard !selectedDrawingIDs.isEmpty 失败静默）|
| ⌘⇧S | v17.166 支撑阻力 toggle | v13.27 画线保存模板（count==1 + !locked 失败静默）|
| ⌘⇧P | v17.164 形态识别 overlay | v17.x 复制截图剪贴板 |

**修法**（commit f5d0f23）：低频 3 个改 ⌘⌥⇧L/S/P · memory `feedback_快捷键防冲突` 加同 file 冲突章节 + 自检 one-liner（commit 前必跑）。

### 3.3 v17.201 真修 · tooltip + Shell 模式 chartMode 入口

**tooltip 真根因**：自定 SwiftUI overlay 显示在按钮上方 32px · Shell 嵌入模式（v17.6+）顶部 PrimaryTabBar 半透明覆盖该区域 → 「模糊白字」。

**修法**：InstantTooltip 改用 macOS 系统 `.help()` · NSWindow 级 · 不被任何 SwiftUI/Shell 覆盖 · 代价 ~1.5s 延迟。

**Shell 模式 chartMode 入口 gap**：v17.6 Shell 模式隐藏 toolbar · Pane Header 不管 chartMode · 整个 Shell 嵌入下无 chartMode 切换 UI · 用户找不到回放。

**修法**：bodyPrefix 加 `shellModeBar` · 仅 chartMode segmented picker · 高 26pt · Shell 模式专用。

### 3.4 v17.202 真修 · CSV 导入 menu Button sheet anti-pattern

**真根因**：`OpenCSVImportButton` 在 CommandMenu 内用 @State + .sheet · **SwiftUI menu Button 不渲染 view tree · sheet modifier 永远不 anchor 弹出 · @State 也不保留**。同模块 OpenFormulaEditorButton 用 `openWindow(id:)` 才工作。

## 5 · 学到的（写入新会话提醒）

### 5.1 同名 @State 跨 struct = 假修复（最大教训）

**v17.190 第 12 波只让 Mac 编译过 · 没真修 state 连通**：
- ChartScene 内 @State + sheet modifier
- ChartContentView 内补同名 @State 让 cannot find error 消失 · 编译过 · 但 state 不连通
- **运行时 sheet 永远不开 / overlay 永不渲染**

**自检**（已加 memory `feedback_Mac_swift63严格三类陷阱` 类型 2 子陷阱）：
- 每次"补 @State 让编译过"前先问：「outer struct 是不是已经有同名 @State」
- 如有 → 改 @Binding · 不要复制 @State
- 「只看编译过判定修复成功」是不够的

### 5.2 SwiftUI menu Button 是 view tree 之外

- CommandMenu 内 Button 仅注册 menu item · 不进 view tree
- 挂 .sheet / .popover / 用 @State 全部无效
- 正确做法：用 openWindow 弹独立 WindowGroup · 同 OpenFormulaEditorButton

### 5.3 同 file 内 keyboardShortcut 也会冲突

不只是跨 window 冲突 · 同 file 同 modifier+key 重复也冲突。SwiftUI 优先靠前的 Button · 后者 guard 失败静默 → 排查极难。

**自检**（已加 memory `feedback_快捷键防冲突`）：
```bash
grep -nE '\.keyboardShortcut\(' Sources/MainApp/ChartScene.swift \
  | sed 's/.*keyboardShortcut(//' \
  | awk -F'modifiers:' '{print $1 "|" $2}' \
  | sort | uniq -c | sort -rn | awk '$1 >= 2 {print "❌ 冲突 x"$1": "$0}'
```

### 5.4 GUI 截图 applescript 盲跑必失败

`osascript keystroke + sleep N + screencapture` 三层不确定相乘 = 失败。改半自动让用户参与确认远胜过盲跑。（已 memory `feedback_GUI截图盲截无效`）

### 5.5 macOS 系统 .help() 比自定 SwiftUI overlay 更稳

任何 chrome / 子 toolbar / Shell PrimaryTabBar 都不会覆盖 NSWindow 级 tooltip。代价是 ~1.5s 延迟。
