### 4.2 v17.207 真修 · PrimaryTab 切换消失 bug

**用户反馈**：5 个一级 tab（看盘/套利/期权/复盘/训练）· 点后面 4 个时 PrimaryTabBar + WorkspaceTabBar 整体消失 · 无法切回。

**Root cause（看 /tmp/debug_primary_tab_bug 截图 + UserDefaults 定位）**：
- Shell VStack 固定 209pt（PrimaryTabBar 32 + WorkspaceTab 28 + BottomBar 120 + StatusBar 26 + 3 Divider）
- 屏幕高 949 - 标题栏 28 = 921pt 可用
- OptionWindow `.frame(minHeight: 720)` → 209 + 720 = 929 > 921
- SwiftUI 强制满足子 view min height → VStack 顶部 PrimaryTabBar/WorkspaceTabBar 切出可见区

**修法（4 个 PrimaryTab.defaultPaneKind 对应窗口）**：
- OptionWindow（minHeight 720 → isHostedInShell ? 0 : 720）+ 加 @Environment isHostedInShell
- SpreadWindow（minHeight 600 → 同）+ 加 @Environment
- ReviewWindow（minHeight 720 → 同）已有 isHostedInShell
- TrainingWindow（minHeight 540 → 同）已有 isHostedInShell

**debug 脚本 v17.207+** `debug_primary_tab_bug.sh`：
- 半自动单张截图 + UserDefaults dump + scp 回 vvsvr
- `--reset` 选项清 shell.v1.primaryTab 让 app 进 default .watching

## 5 · 学到的（已写入 memory）

### 5.1 补丁累积叫停（新加 memory · 永久）

`feedback_补丁累积叫停.md`：同一架构问题打到第 3-5 个补丁时 AI 必须主动叫停做架构级回顾 · 不等用户来问。

v17.0+ 累计 6+ Shell 补丁后用户来问才反思 · 这是 AI 责任 · memory 全是鼓励主动反思的指令。下次：v17.6 第 1 个补丁合理 · v17.196 第 3 个开始警觉 · v17.201 第 4 个就该叫停 · 不等到 v17.207 第 6 个。

### 5.2 业界架构选型经验

- SwiftUI 单一 view tree 不适合多面板嵌入大型独立窗口（子 view min/max 穿透撑父）
- AppKit NSSplitViewController + NSHostingController 是业界标准（Xcode / Photoshop / Sketch）
- NSPopover 替代 SwiftUI overlay tooltip（0 延迟 · 不被遮）
- NSPanel 替代 SwiftUI sheet 长期工具窗（不抢焦点 · floatLevel 浮顶）

### 5.3 大型架构决策前必 spike

50-100 行 prototype 1-2h 验证可行性 · 比凭直觉决策可靠 100 倍。
