#### v17.223 真修 · isInV1MainWindow + ChartScene minWidth=0

**真根因**：ChartScene `.frame(minWidth: 800, minHeight: 480)`（line 424 + line 3033）·在 NSHostingController 内会撑大 view·**视觉上覆盖 NSSplitView divider 区域**·mouse event 被 ChartScene 内部 gesture 抢走·NSSplitView 拖不动。

ShellSidebar / WatchlistWindow（嵌入单栏模式）没设 minWidth · 不撑大 · 不干扰。

修法：
- AppKitShellEnvironment 新增 `IsInV1MainWindowKey` EnvironmentKey
- injectAppKitShellEnv 注入 `isInV1MainWindow=true`
- ChartScene + ChartContentView 加 `@Environment(\.isInV1MainWindow)` · frame 处条件化：
  - V1 主窗内（isInV1MainWindow=true）· minWidth=0 minHeight=0 · 不撑大
  - 独立 chart 窗（chart WindowGroup id "chart"）· isInV1MainWindow=false · minWidth=800 不变

isInV1MainWindow 与 isHostedInShell 区别：
- isHostedInShell · V1 实际语义=isInMonitorPanel · 仅 wrapAsMonitor 注入 · 控制 WatchlistWindow 等单栏/双栏 layout
- isInV1MainWindow · V1 所有嵌入子组件注入 · 控制 minWidth/minHeight 避免撑大 NSSplitView divider 覆盖
