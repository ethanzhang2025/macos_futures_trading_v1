### 4.1 元问题深度反思（用户质疑）

用户问"为什么 Mac 端这么多 bug 都难修"。我做工程根因分析：
- 反馈链严重不对称（Linux 改 → Mac 验 · 5-30 分钟/轮）
- 没有 isolated minimal repro
- 架构选型本身有缺陷（NSSplitView 多层嵌套做 grid）
- "猜根因"模式没强制诊断
- AppKit 知识深度不够
- Linux/Mac 环境差距（build pass 不等于行为正确）

**结论**：不是技术不可突破 · 是工程方法要改。

### 4.3 Appium Mac2 UI 测试自动化闭环（核心成果）

**架构**：
```
[Linux] Claude 改 → push
[Mac] git pull && ./mac_loop.sh  ← 唯一手动操作
   1. 环境自检 + venv 自动 setup
   2. (用户已 pull)
   3. ./run-mac.sh debug build → MainApp.app
   4. 启 appium server
   5. pytest ui_tests/generated/
   6. 收集 result.json + shots + ui_tree + logs
   → 停 appium (v17.263)
   7. scp → vvsvr
   8. graceful 关 MainApp (v17.258)
[Linux] ./pull_results.sh → /tmp/appium_loop/
   读 result.json + 看 shots/*.jpg + 读 ui_tree_*.xml
```

**文件清单**（10+ 新建）：
- `mac_loop.sh` 230 行 · Mac 端 8 步全流程
- `pull_results.sh` 82 行 · Linux 端拉结果
- `ui_tests/README.md` 135 行 · 工作流文档
- `ui_tests/requirements.txt` · appium-python-client + pytest + Pillow
- `ui_tests/conftest.py` · pytest session driver fixture
- `ui_tests/lib/helpers.py` · click_menu_item / press_keys 整数 modifier / find_window_by_id / dump_ui_tree
- `ui_tests/specs/5.1_四宫格divider拖动.md` · 用户写的 markdown 测试说明
- `ui_tests/generated/test_step_00_smoke.py` · 启动验证
- `ui_tests/generated/test_step_01_open_v1.py` · 切 V1 主窗

**关键技术决策**：
- 测试目录 `ui_tests/` 不用 `tests/`（防 macOS case-insensitive 与 Swift `Tests/` 合并）
- conftest.py 放 `ui_tests/` 根目录（不放 lib/ 子目录 · pytest 自动加载）
- modifier 用 NSEventModifierFlags 整数（MOD_CMD_CTRL = 0x140000）· 不用字符串
- 菜单导航走 `click_menu_item(driver, *path)` · XPath 按 `@title` 匹配 · ESC 关 popup
- 截图压缩 JPEG q=75 + 1600px（Pillow）· 1.5MB → 175KB
- step-by-step 模式（test_step_00 / 01 ...）· `--until N` 渐进推进
- cleanup 顺序：`stop_appium()` → 2s 等 → `quit_main_app()`（SIGTERM）
- mac_loop 早停 appium：[6/8] 收集完立即停 · scp 期间不占资源
- Pillow / Python venv / appium driver 全部 mac_loop 首次跑自动 setup

### 4.4 5.1 节 3 个真 bug · 全部修复

**Bug A · monitor emoji section header 不显示**（v17.260）
- ui_tree 揭示 ⭐️🗂💼 真渲染了 · frame 完整
- root cause = 样式太淡（size 11 + .secondary 灰）
- 修法：14pt + .bold + .primary + accent 渐变背景 + 边线

**Bug B · chart toolbar 不渲染**（v17.261-264）
- ui_tree 揭示 ScrollView width=200 但内容 1450pt · 装不下
- root cause = chart pane 太窄 · toolbar 设计 1450pt 不适合
- 修法：
  - v17.261 新增 `v1MainModeBar` 精简栏 · 仅 chartMode + symbol + period
  - v17.262 Text `.fixedSize` 防 Spacer 压成 width=1
  - v17.264 删 chartMode segmented · 简化为 symbol + period（避免超界）

**Bug C · divider 拖不动 / center 不变**（v17.265-266）
- 用户报告：左右 divider 能拖 · 但中间四宫格不变
- root cause 双层：
  1. `holdingPriority` 默认 250 三个相同 · NSSplitView 把 delta 推给 monitor
  2. `centerItem.minimumThickness=400` 卡在最小值 · center 不能再 shrink
- 修法：
  - v17.265 设 holdingPriority（sidebar 252 / center 248 / monitor 252）
  - v17.266 centerItem.minimumThickness 400 → 200（让 center 真正能伸缩）
