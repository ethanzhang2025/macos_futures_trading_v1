## 1. 项目一句话 + 快速了解

### 项目是什么
**为专业交易者打造的 Mac/iPad 原生专业工作台** —— 盯盘、预警、复盘、训练、日志，让主交易软件继续用，把工作流放到 Mac 上。中国期货版的 **TradingView × Linear × iPad Pro**。

### 为什么做
中国期货 250 万活跃账户中 Mac 渗透率约 8%（≈20 万人），但主流终端（文华/博易/快期/易盛）全是 Windows MFC/Delphi 老架构，UI 停留在 2010 年代。**85% 以上 Mac 用户被迫装 Parallels 跑文华**。Apple Silicon + SwiftUI + iPad Pro 形态为品质优先的 Mac 原生产品打开窗口。

### Vision / Mission
- **Vision**：中国期货版的 TradingView × Linear × iPad Pro 专业工作流
- **Mission**："让 Mac 用户在中国期货市场，拥有一款超一流的原生交易工具。"

### 核心定位澄清
- **Stage A（M0-M9）**：专业工作台 · **不接 CTP 下单** · 活下来
- **Stage B（M12-M24）**：加 CTP 下单 + 麦语言完整兼容 + iPad 专业工作流 · 做大
- **Stage C（M24+）**：据结果定方向（Windows / 证券 / 出海 / 策略市场）

## 9. 北极星 + 产品原则

### 北极星指标：WAPU
Weekly Active Pro Users · 过去 7 天内至少打开 3 次的 Pro 订阅用户数
**每天 9am 开盘前看昨天 WAPU · 连续 7 天持平或下降 → 启动死亡率自检会议**

### 产品原则 7 条（决策裁决书）
1. **Ship weekly, polish later** · 周五必发
2. **Charge early, be transparent** · 能收费就早收费
3. **Fast is a feature** · < 100ms / < 1s / 60fps
4. **Mac-native aesthetic, not Mac-only lock-in** · UI 原生逻辑跨平台
5. **Keyboard is first-class** · 全键盘可达
6. **Multi-device is multi-context** · iPad/iPhone 专属布局
7. **Respect traders' money and attention** · 稳定 > 创新

**冲突时优先级**：**7 > 3 > 2 > 1 > 其他**（稳定 > 速度 > 现金 > 节奏）

## 17. 项目主目录改名问答

### 能改名吗
**可以**。但改名会影响：

1. **Claude Code 会话历史** · 按路径 hash 存储，改名后 `claude -c` 找不到旧会话 → 丢失对话历史（但本交接文档已捕获关键决策）
2. **Memory 文件** · 存在 `~/.claude/projects/-home-beelink-macos-tmp-view-cc-usaged/memory/` 下，改名后新会话不会识别
3. **本项目文档内的路径引用** · 少数文档提到具体目录（大部分用相对路径或文件名，影响小）

### 改名的 3 种做法

**选项 A · 干净改名（推荐）**
```bash
# 1. 退出 Claude Code
# 2. 改名项目目录
mv /home/beelink/macos_tmp/view_cc_usaged /home/beelink/macos_tmp/期货终端项目
# 3. 改名 Claude 项目数据目录
mv ~/.claude/projects/-home-beelink-macos-tmp-view-cc-usaged \
   ~/.claude/projects/-home-beelink-macos-tmp-期货终端项目
# 4. 启动新会话（基于本交接文档无缝继续）
cd /home/beelink/macos_tmp/期货终端项目
claude
```
- 优点：干净、彻底、名字合意
- 缺点：本次会话历史丢失（但无所谓，关键决策在交接文档里）

**选项 B · 保留旧名字，加软链接**
```bash
cd /home/beelink/macos_tmp/
ln -s view_cc_usaged 期货终端项目
# cd 期货终端项目 进入时实际还是 view_cc_usaged
```
- 优点：零风险，Claude 内部路径不变
- 缺点：ls 还能看到旧名字

**选项 C · 继续用旧名字**
- 最简单，但 view_cc_usaged 这个名字确实不合适项目

### 推荐
**改名（选项 A）**。因为：
- 交接文档已经完整捕获项目所有关键信息
- 新会话读交接文档即可无缝接上
- 未来 Git 仓库、Cursor 项目等都会沿用新名字
- 旧会话历史价值有限（决策已进文档）

**建议的新名字**（可选）：
- `期货终端项目` · 最直白
- `mac-futures-terminal` · 英文版（但违反中文命名规则，除非目录名算特例）
- `中国期货Mac终端` · 更精确
