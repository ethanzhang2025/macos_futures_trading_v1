## §2 · 候选人画像

### 2.1 产品 / Pro 用户顾问 · 必备

- **交易经验**：≥3 年期货实盘（含商品 + 金融期货）
- **当前主力工具**：文华 / 快期 / 同花顺期货端 / 易盛之一
- **痛点共鸣**：曾抱怨过"Mac 端没好工具"或"麦语言碎片化"
- **可投入时间**：每周 4-6 小时（评审 1 小时 + 试用反馈 2-3 小时 + 异步沟通 1-2 小时）
- **沟通**：愿意中文异步 + 必要时视频 1on1
- **加分项**：在雪球 / 知识星球 / 期货吧有公开发声

### 2.2 设计顾问 · 必备

- **作品集**：≥2 个上架 Mac App Store 的原生应用（≥4.5 星 + ≥1k 评价优先）
- **熟悉**：SwiftUI + Apple HIG + Sketch/Figma + Mac 信息架构（不止移动端经验）
- **审美**：偏 Linear / Things / Tower / Reeder 一类（vs 跨平台 Electron 的反例）
- **可投入时间**：每周 4-6 小时
- **加分项**：苹果 ADA 获奖 / Sidebar.io 推荐作品 / 独立工作室合伙人

### 2.3 通用要求（所有顾问）

- 不在竞品公司任职（参考合伙协议 §25 竞业范围）
- 愿意签 NDA + IP 协议（合伙协议 §27 配套）
- 接受期权激励（不要求现金报酬）
- 大方分享真实意见（不阿谀）

## §4 · 期权激励

### 4.1 期权池总览（合伙协议 §6）

- **总额**：10%（公司全部股权的 10%）
- **代持人**：甲方
- **管理**：任何授予需双方书面同意

### 4.2 顾问期权额度（典型）

| 顾问类型 | 单人期权额度 | vesting | 加速条款 |
|---------|------------|---------|---------|
| 产品顾问（核心 1 位） | **0.5% - 1%** | 4 年 + 1 年 cliff | 公司被收购 single trigger |
| 设计顾问（核心 1 位） | **0.5% - 1%** | 4 年 + 1 年 cliff | 同上 |
| 行业顾问（按需 1 位） | 0.25% - 0.5% | 同上 | 同上 |
| 增长 / GTM 顾问 | 0.25% - 0.5% | 同上 | 同上 |
| 关键外包朋友（Metal / CTP 等） | 0.5% - 2% + 现金 | 3 年 + 6 月 cliff（短跑道）| 同上 |

### 4.3 期权池分配规划（建议）

```
期权池总额 10% =
   ├─ 顾问层（4 类 × 0.5-1%）         ≈ 2.5-4%
   ├─ 关键外包（4-6 人 × 0.5-1.5%）  ≈ 3-6%
   └─ 后续核心员工（M6+）           ≈ 2-3.5%
```

**保守原则**：M3 末前累计授予 ≤4%，留 ≥6% 给后续核心员工 + Stage B 拓展。

### 4.4 期权授予协议（独立文档）

每位顾问 / 外包朋友须签：

- **期权授予协议**（Option Grant Agreement）
- **NDA**（保密协议）
- **IP 转让协议**（个人贡献的代码 / 设计归公司）
- **顾问服务协议**（角色 / 时长 / 退出机制）

模板由律师出具（合并 WP-02）。

### 4.5 vesting 跟踪

- 顾问与外包均按 4 年 vesting + 1 年 cliff（外包朋友可短跑道版 3+0.5）
- 每月 standup 时同步 vesting 进度
- 飞书文档"期权登记表"维护（敏感权限）

## §5 · 评审节奏

### 5.1 周度评审（每位顾问独立）

- **时间**：与该顾问约定固定时段（建议周三或周五下午）
- **时长**：30-60 分钟
- **形式**：飞书视频
- **议程**：详 §6 评审议程模板

### 5.2 月度联合评审（M3 末起启用）

- **时间**：每月最后一周（与 standup 同周）
- **参与**：你 + Hunter + 全部顾问
- **时长**：60-90 分钟
- **目的**：跨顾问对齐 + 共识共建

### 5.3 季度顾问 review（你单边评估）

- **时间**：M3 / M6 / M9 末
- **目的**：评估顾问贡献 + 调整期权 / 续约 / 退出
- **机制**：详 §8

## §6 · 周度评审议程模板

### 6.1 产品顾问议程（30-45 min）

```
[周三 / 周五 14:00]

1. 上周本人试用反馈（顾问主导，10 min）
   - 用过哪些新功能
   - 卡点 / 槽点（具体场景）
   - 与文华 / 快期对比

2. 产品方向 / 路线图同步（你主导，10 min）
   - 本周完成 WP / 下周计划
   - 重大决策征询

3. 路线图调整建议（顾问主导，10 min）
   - 优先级排序意见
   - 用户反馈洞察

4. 行动项（5 min）
   - 顾问下周需试用什么
   - 你下周需调整什么
```

### 6.2 设计顾问议程（30-45 min）

```
[周三 / 周五 16:00]

1. 上周设计稿 review（顾问主导，15 min）
   - 视觉规范一致性
   - 信息架构合理性
   - Apple HIG 遵循度

2. 性能 / 实现 trade-off 讨论（你 + 顾问，10 min）
   - 设计 vs 工程平衡

3. 下周设计需求（你主导，10 min）
   - 优先级 + 交付时点

4. 行动项（5 min）
```

### 6.3 议程灵活原则

- 没什么可聊时（≤2 个话题）：直接缩短到 15 分钟
- 重大议题（≥4 个话题）：延长到 60 分钟
- 不为开会而开会

## §8 · 顾问续聘 / 调整 / 退出

### 8.1 季度评估（M3 / M6 / M9）

| 评估维度 | 权重 | 评分 |
|---------|------|------|
| **可投入时间稳定性**（每周 4-6 小时是否达成）| 30% | 1-5 |
| **洞察价值**（提出多少被采纳的建议）| 30% | 1-5 |
| **响应速度**（异步沟通 / 1on1 守时）| 20% | 1-5 |
| **关系健康度**（合作愉快度）| 20% | 1-5 |

### 8.2 评估结果处理

| 总分 | 处理 |
|------|------|
| ≥4.0 | 继续合作 + 期权按 vesting 解锁 + 可考虑增加额度（最多 +0.5%）|
| 3.0-3.9 | 续约但 1on1 谈话 + 调整议程 / 范围 |
| 2.0-2.9 | 警告 + 1 个月观察 / 改进 |
| <2.0 | 终止合作（按 §8.3 退出条款）|

### 8.3 顾问退出条款（合并到顾问服务协议）

- **主动退出**：提前 30 天书面通知；按 vesting 实际解锁 + 已 vested 部分按公允价值回购或保留（双方协商）
- **被动退出**（评估不达标）：终止时已 vested 期权按 ¥0 回购至期权池（与 Bad Leaver 类似但更宽容）+ 未 vested 部分公司收回
- **不可抗力退出**：按 Good Leaver 处理（已 vested 按公允价值回购）

### 8.4 利益冲突触发即终止

- 顾问加入竞品（含咨询）→ 立即终止 + 期权全部收回 + 触发 NDA / 竞业违约金（同合伙协议 §35）
- 顾问泄露源代码 / 用户名单 → 同上 + 法律追责
