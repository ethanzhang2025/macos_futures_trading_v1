## §3 · 沟通节奏

### 3.1 日常异步（默认）

- **工具**：飞书（首选）/ 微信（备用，仅紧急）
- **响应预期**：工作时段（周一至周六 9:00-22:00）2 小时内
- **不可中断时段**：你的"产品时间"（每天预留 2 小时不被打扰；推荐 14:00-16:00 或 20:00-22:00 任选）

### 3.2 周度同步（每周 1 次）

- **形式**：1on1（飞书视频或当面）
- **时长**：30-45 分钟
- **时间**：每周一上午 10:00（雷打不动；缺席自动顺延至次日）
- **议程模板**：
  1. 上周回顾：你做了什么 / Hunter 做了什么（各 5 分钟）
  2. 数据回顾：本周 TestFlight 新增 / 触达数 / 反馈 highlights（5 分钟）
  3. 本周计划：你的工程 WP / Hunter 的外联目标（10 分钟）
  4. 阻塞 + 求助（10 分钟）
  5. 关键决策与共识（5 分钟，归档至飞书）

### 3.3 月度 Standup（每月 1 次 · WP-12 详）

- **时间**：每月最后一个周五下午 15:00
- **时长**：15 分钟（严格控时）
- **目的**：死亡率自检（详 WP-12）
- **不开大会、不写报告、不纠结**

### 3.4 季度 Review（每 3 个月 1 次）

- **时间**：M3 / M6 / M9 末（与 Stage A 三大 checkpoint 对齐）
- **时长**：2-3 小时（半天）
- **目的**：阶段性回顾 + 下季度战略校准
- **议程**：
  1. 季度数据全景（WAPU / 留存 / 付费 / 跑道）
  2. 路线图 vs 实际进度差异分析
  3. 风险全景表更新（WP-132）
  4. 战略调整：是否需要砍范围 / 改方向 / 启动 Plan B（详 D3 §2-3）
  5. 下季度目标设定 + 个人 OKR 对齐

### 3.5 紧急沟通（事故 / 重大决策）

- **触发**：生产事故 / 监管变化 / 资金底线告警 / 合伙人意外 / 任一指标连续 2 月红灯
- **响应**：电话或微信视频；30 分钟内召集
- **后续**：24 小时内出会议纪要（飞书归档）

## §5 · 决策流程

### 5.1 日常事项（按 RACI 矩阵）

- 谁是 A（拍板者）就由谁决定
- 协商不成时按合伙协议 §13 处理（股权比例 1:1 投票，甲方天然多数）
- 决策记录于飞书文档 + 月度 standup 时回顾

### 5.2 重大事项（合伙协议 §14）

由甲方单方决定 + 提前 30 天书面通知乙方 + 乙方反对披露权：

- 引入新合伙人 / 融资
- 公司被收购 / 出售核心资产
- 公司解散 / 清算
- 修改本协议或合伙协议核心条款
- Stage B 启动决策（M9 末）
- 战略转型
- 单笔 ≥¥20,000 的非常规支出
- 关停或重大调整核心模块

### 5.3 决策升级机制

| 触发 | 升级路径 |
|------|---------|
| 日常事项双方僵持 ≥48 小时 | 升级到下次 1on1 讨论 |
| 重大事项乙方提反对意见 | 甲方 7 天内书面回应 → 仍不一致进入飞书会议纪要归档 |
| 资金 / 合规事项有疑虑 | 顾问咨询（兼职顾问）→ 律师咨询（WP-02）|

## §6 · 工作时长 + 远程协作约定

### 6.1 工作时段

- **默认**：周一至周六 9:00-22:00（不强制坐班；自主安排）
- **核心重叠时段**：每天 14:00-18:00（保证双方在线 ≥4 小时）
- **周日**：弹性休息（紧急除外）
- **法定节假日**：休息（但 App Store 紧急事故除外）

### 6.2 远程办公

- **办公地点**：双方各自家中或自选场所
- **设备**：双方各自承担（公司报销关键工具：Mac Studio / 显示器 / 软件订阅，详财务报销 SOP）
- **会议形式**：飞书视频默认；一线城市同城时每月当面 1 次

### 6.3 请假与休息

- **短假（≤3 天）**：飞书通知对方即可
- **长假（>3 天）**：提前 1 周通知 + 交接关键事项
- **病假 / 急事**：尽快电话或微信通知
- **不可抗力（连续 ≥7 天）**：触发合伙协议 §29-32 创始人意外条款

## §7 · 知识管理

### 7.1 飞书文档目录结构（标准）

```
Stage A · 工作根目录/
├── 00 · 战略文档/（D1-D6 / 产品设计书 / 补遗）
├── 10 · 工作包清单/（Stage A WP / 映射表 / ChatGPT 包）
├── 20 · 决策记录/（按月归档；含 1on1 / standup / 重大决策）
├── 30 · 用户访谈/（M1-M3 主战场）
│   ├── 候选名单 / 邀约话术 / 访谈大纲
│   └── 访谈记录（每场 1 文档：受访者 / 时间 / 关键洞察 / Action）
├── 40 · 销售外联/（Hunter 主战场）
│   ├── 渠道 SOP（雪球 / 知识星球 / 微博等）
│   ├── 触达数据日志（每天 / 每周）
│   └── 销售话术库
├── 50 · 客服与 VIP 群/（M5+ 启用）
├── 60 · 财务对账/（月度账目 / 月薪记录 / 业绩奖金计算）
├── 70 · SOP 库/（关键操作流程 / 紧急响应 SOP）
├── 80 · 顾问评审/（每周顾问反馈 / 期权进度）
└── 90 · 风险与生死自检/（看板 / 风险全景表 / Plan B 触发记录）
```

### 7.2 沉淀原则

- **写比说重要**：所有决策必须落到飞书或 GitHub commit
- **可检索 > 完美**：先记录，再润色
- **30 天 review**：每月 standup 时检查文档死链 / 过期内容
- **公开默认**：飞书文档默认双方可见（敏感数据如薪资 / 期权细节单独权限）

## §8 · 利益冲突与边界

### 8.1 双方个人活动

- **甲方**：本协议生效期间不得参与其他全职工作；个人技术博客 / 开源贡献需告知乙方
- **乙方**：本协议生效期间不得参与其他全职工作；销售类副业（佣金制）需经甲方书面同意

### 8.2 第三方咨询 / 外包

- 甲方接外包技术咨询：单笔 ≥¥5,000 需告知乙方
- 乙方接外包销售咨询：单笔 ≥¥3,000 需告知甲方
- 双方均不得为竞品方咨询（详合伙协议 §25 竞业限制）

### 8.3 资源使用

- 公司资金（¥20w 启动金 + 后续订阅收入）专用于公司经营
- 个人时间分配：双方有义务保证每周 ≥40 小时投入公司事务

## §9 · 协作约定的生效与修订

### 9.1 生效

- 本约定自双方签字（电子签或飞书已读确认均可）之日起生效
- 与合伙协议（WP-10）冲突时，以合伙协议为准
- 与公司章程冲突时，以法律强制性规定为准

### 9.2 修订

- **小修（单条措辞 / 工具升级 / 节奏微调）**：双方书面同意 + 飞书文档版本号 +1
- **大修（角色定位 / RACI 主辅责 / 决策流程）**：按合伙协议 §14 重大事项处理（甲方单方决定 + 30 天通知 + 乙方反对披露权）
- **季度 review 时统一回顾**：M3 / M6 / M9 末季度 review 必须包含本约定 review

### 9.3 协议存续

- 本约定有效期：Stage A 全周期（M0-M9）
- M9 末季度 review 时启动 Stage B 版本重写
