## §1 · 设计取舍

### 1.1 为什么是 15 分钟（不是 60 分钟）

**原方案**：5 个指标 + 月度正式会议
**最终方案**：**2 个指标 + 15 分钟月度 standup**（D3 §1）

**理由**：
- 两人团队开销敏感，避免为自检而自检
- 2 个指标已足够捕捉 Stage A 最关键信号
- 自检 ≠ 终止讨论；自检 = **方向修正触发器**
- 真有问题 → 升级到 Plan B 讨论会（独立 1-2 小时会议）

### 1.2 15 分钟的硬约束

- 不开大会 / 不写报告 / 不纠结
- 不展示 PPT / 不准备发言稿
- 一张飞书表格 + 双方实时核对 + 当场出结论

## §2 · 北极星指标 · WAPU

### 2.1 定义（D1 §4）

**WAPU** = Weekly Active Pro Users

**= 过去 7 天内至少打开产品 3 次的有效期内 Pro 订阅用户数**

### 2.2 为什么是 WAPU

- 一个数字同时覆盖**付费（商业价值）× 活跃（用户价值）**
- 过滤"买了不用"的僵尸付费
- 与 Stage A → B → C 阶段一致（无需切换北极星）

### 2.3 阶段对照（D1 §4）

| Stage | 北极星 | 辅助 1（领先指标） | 辅助 2（滞后验证） |
|-------|-------|-------------------|------------------|
| **Stage A** | WAPU | TestFlight 周新增 → Pro 转化率 | M1 留存率（>70% 健康）|
| Stage B | WAPU | 多端活跃占比、下单频次 | NRR 净收入留存（>100% 健康）|
| Stage C | WAPU | 新市场（券商/证券/海外）渗透率 | LTV/CAC > 3 |

### 2.4 每日使用 SOP（创始人主导）

- 早上 9 点开盘前：看昨天的 WAPU
- 数字掉了：问为什么（bug？推送出错？行情异常？）
- 数字涨了：问哪来的（哪个渠道？哪个功能触发？）
- **连续 7 天持平或下降 → 启动死亡率自检会议**（不等月度 standup）

### 2.5 数据来源

| 阶段 | 数据来源 | 维护成本 |
|------|---------|---------|
| **M1-M3**（埋点未就绪 / Pro 未上线）| 手动估算 + TestFlight 反馈 + Hunter 触达数据 | 低 |
| **M3-M5**（埋点就绪 / TestFlight 阶段）| WP-133 埋点 + SQL 查询 | 中 |
| **M5+**（Pro 订阅上线）| 自动 Dashboard（飞书或 Metabase）| 低 |

## §4 · 月度 Standup SOP

### 4.1 基础信息

| 项 | 值 |
|---|---|
| **时间** | 每月最后一个周五下午 15:00 |
| **时长** | 15 分钟（严格控时；超过 20 分钟拆 Plan B 讨论会）|
| **参与** | 你 + Hunter |
| **形式** | 飞书视频（同城时可当面）|
| **主持** | 你 |
| **记录** | Hunter（飞书文档实时录入）|

### 4.2 议程模板（15 分钟）

```
[0:00 - 3:00] 开场 + WAPU 看板（3 分钟）
  - 本月 WAPU 数字
  - 与上月对比 / 趋势

[3:00 - 8:00] 指标 1：TestFlight 周新增（5 分钟）
  - 本月实际值 vs 目标 → 红 / 黄 / 绿
  - 红 / 黄：上月试了 A 没效果，下月试 B（具体写出）
  - 绿：哪个渠道触发？是否可加倍？

[8:00 - 12:00] 指标 2：累计 Pro 付费数（4 分钟）
  - 本月新增付费 X
  - 累计付费 Y
  - 偏差 Z（vs 节点目标）

[12:00 - 14:00] 风险全景表 review（2 分钟，WP-132 联动）
  - 11 项风险有无新触发信号
  - 任何升级 / 降级

[14:00 - 15:00] 行动项 + 结束（1 分钟）
  - 下月 1-3 个具体动作
  - 不开大会、不写报告、不纠结
```

### 4.3 时间控制规则

- 主持人手机定时器：12:00 / 14:00 / 15:00 三档提醒
- 超时议题立即扔到 Plan B 讨论会或下次 1on1
- 任何 ≥3 分钟的争论 → 强制收口 + 异步飞书继续

## §6 · Plan B 讨论会（触发后）

### 6.1 触发条件（任一）

- TestFlight 周新增连续 2 月红
- Pro 付费数连续 2 月红（M6 起适用）
- WAPU 连续 7 天下降（创始人单边触发）
- 重大风险升级（合伙人持续力不足 / 跑道告急 / 监管突变 等）

### 6.2 议程（1-2 小时）

```
[0:00 - 0:15] 数据复盘（15 分钟）
  - 哪个指标红？多久了？
  - 哪些假设失效了？

[0:15 - 0:45] 方向 vs 方法判断（30 分钟）
  - 方向红：是否要砍范围 / 砍 Persona / 改 GTM
  - 方法红：是否换渠道 / 换话术 / 加预算

[0:45 - 1:30] Plan B 选项评估（45 分钟，D3 §2-3）
  - 砍范围（先砍 iPad / 先砍麦语言部分函数）
  - 改渠道（停雪球 / 试少数派 / 试垂直社群）
  - 接外包补血（你 / Hunter 接技术外包延长跑道）
  - 重新考虑小额天使（最后选项）

[1:30 - 2:00] 行动项 + 时点（30 分钟）
  - 下月 1-3 个具体调整
  - M3 / M6 checkpoint 时再 review
```

### 6.3 输出

- Plan B 决策文档（飞书归档 · 90 · 风险与生死自检 / 目录）
- 调整后的 Stage A 工作包优先级
- 是否触发 D3 §2 资金底线告警（¥12w 黄线 / ¥9w 应急 / ¥3w 硬停）
